# 一、核心方法论：种子创作法

## 1.1 什么是种子创作法

```
【输入】用户给一个"种子"
  ├── 核心情节（被背叛/被欺负/被冤枉等）
  ├── 人物关系（夫妻/闺蜜/婆媳/师徒等）
  └── 创意点（独特设定/社会矛盾/人物反差等）

【输出】AI发散变体
  └── 同一情节 → 不同年代/职业/身份/金手指的多个版本

【本质】固定故事结构 + 变量自由组合 = 高效批量创作
```

## 1.2 为什么用种子创作法

| 对比 | 固定模板 | 种子创作法 |
|------|---------|------------|
| 用户输入 | 少，但固定 | 中等，但灵活 |
| 创意独特性 | 中等（套公式） | 高（每次发散不同） |
| 批量生产 | 一般 | 强（一个种子出多个变体） |
| 爆款率 | 依赖模板通用性 | 依赖种子质量+发散创意 |
| 适配平台 | 一般 | 强（可针对不同平台调优） |

## 1.3 核心公式（固定结构）

所有短故事都遵循7阶段结构，每个阶段的反转节点和伏笔要求必须标注：

```
阶段1【开场冲突】(1-3章)
  └── 被背叛/被欺负/被冤枉 → 情绪钩子
  └── 【反转节点】此阶段至少1个反转（如：看似坏人其实是好人/看似帮助实际是陷害）
  └── 【伏笔埋入】至少1个伏笔（如：配角的奇怪行为/物品/话语）

阶段2【重生/穿越】(4-5章)
  └── 获得第二次机会 + 发现金手指（可选）
  └── 【反转节点】可选（如：重生后发现事情并非表面那样）
  └── 【伏笔回收/新埋】回收阶段1的伏笔，并埋入新伏笔

阶段3【小试牛刀】(6-10章)
  └── 第一次反击成功 → 建立读者信心
  └── 【反转节点】此阶段至少1个反转（如：以为胜券在握实际落入圈套/看似绝境实际转机）
  └── 【伏笔埋入】至少1个新伏笔（为后续反转铺垫）

阶段4【中段积累】(11-20章)
  └── 复仇准备 + 发现更深阴谋 + 势力积累
  └── 【反转节点】此阶段至少1个反转（如：盟友变敌人/敌人变盟友）
  └── 【伏笔埋入】至少1个新伏笔（核心伏笔在此阶段深化）

阶段5【大高潮前奏】(21-25章)
  └── 最终对决前夕 + 感情线确定
  └── 【反转节点】此阶段至少1个反转（如：反派露出真面目/意外强援出现）
  └── 【伏笔回收】回收至少2个伏笔（为高潮蓄力）

阶段6【大高潮】(26-30章)
  └── 终极打脸 + 反派认罪 + 复仇完成
  └── 【反转节点】此阶段至少1个大反转（如：主角反杀/反派阴谋曝光/隐藏身份揭示）
  └── 【制胜逻辑】主角胜利必须基于智慧或能力，禁止"反派降智送人头"
  └── 【伏笔回收】全部伏笔在此阶段回收完毕

阶段7【HE结局】(31-35章)
  └── 甜蜜收尾 + 身份揭示 + 完美落幕
  └── 【最终反转】可选（如：彩蛋式反转为续作埋伏笔）
```

**伏笔规则：** 全部伏笔必须在阶段6结束前回收完毕，不得有悬空伏笔。

# 二B、反派设计模块（必须遵循）

## 二B.1 反派设计三要素

每个反派必须同时具备以下三要素，缺一不可：

| 要素 | 定义 | 示例 |
|------|------|------|
| **动机** | 反派做坏事的原因，必须合理可理解 | "我也曾被伤害过" / "为了保护家人" / "出身低微受尽欺压" |
| **目标** | 反派想要达成的目的，必须清晰 | "取代主角地位" / "夺取家族权力" / "复仇" |
| **策略** | 反派实现目标的手段，必须有脑子 | "借刀杀人" / "挑拨离间" / "设局陷害" |

**反面案例（禁止）：**
```
❌ 反派就是坏，看主角不顺眼就欺负他（无脑坏）
❌ 反派智商下线，主角轻松获胜（降智反派）
❌ 反派实力忽强忽弱，看情况需要（战力崩塌）
```

**正确案例：**
```
✅ 反派A：动机（曾被主角家族迫害）→ 目标（毁掉主角家族）→ 策略（先拉拢再背叛+利用主角弱点设局）
✅ 反派B：动机（出身底层想翻身）→ 目标（夺取公司控制权）→ 策略（潜伏多年收集把柄+安插内鬼+关键时刻一击致命）
```

## 二B.2 反派等级设计

| 反派等级 | 出现阶段 | 策略复杂度 | 主角应对难度 |
|---------|---------|-----------|-------------|
| **小反派** | 阶段1-3 | 单线策略，单个手段 | 低，主角凭金手指可应对 |
| **中反派** | 阶段3-5 | 多线策略，会利用盟友 | 中，主角需要智取+成长 |
| **大反派** | 阶段5-6 | 复合策略，多层布局 | 高，主角必须用智慧+能力+伏笔联动才能战胜 |

## 二B.3 主角制胜逻辑约束

**核心原则：主角胜利必须基于智慧或能力，禁止"反派降智送人头"。**

| 制胜类型 | 说明 | 示例 |
|---------|------|------|
| **智慧型** | 主角利用信息差/布局/识破反派计谋获胜 | 主角早知反派会背叛，提前收集证据反杀 |
| **能力型** | 主角金手指/专业能力在关键时刻发挥作用 | 神医空间关键时刻救活关键人物 |
| **组合型** | 智慧+能力双重胜利 | 主角识破陷阱+用能力反制（推荐） |

**禁止出现：**
```
❌ 反派突然变蠢，自己犯低级错误
❌ 反派临阵倒戈，毫无铺垫
❌ 主角靠"运气好"获胜
❌ 反派强行降智，为了让主角赢而输
```

## 二B.4 场景功能角色（非强行添加）

**路人/工具人原则：按需出现，服务剧情，不得强行添加。**

| 角色类型 | 出现场景 | 功能 | 示例 |
|---------|---------|------|------|
| **场景过渡** | 主角到新地点 | 交代环境/引出剧情 | 茶馆偶遇/街头目睹 |
| **信息传递** | 需要传递情报 | 透露关键信息 | 丫鬟传话/朋友告知 |
| **冲突推动** | 需要制造矛盾 | 推动情节发展 | 绿茶闺蜜煽风点火 |
| **打脸工具** | 需要爽点场景 | 提供打脸机会 | 势利眼亲戚/拜高踩低同事 |
| **背景板** | 群戏场景 | 营造氛围 | 婚礼宾客/宴会人士 |

**检查标准：** 每增加一个路人角色，必须能回答"这个角色删掉会影响剧情吗？"如果不影响，不加。

# 三、发散变体方法论

## 3.1 变体维度矩阵

```
种子情节 + 年代 + 身份 + 职业 + 关系 + 金手指 + 社会矛盾
     ×         ×        ×       ×        ×          ×
   固定      可变    可变     可变     可变        可变
```

## 3.2 发散步骤

```
Step 1: 确定种子（用户输入）
  └── 核心情节 + 人物 + 关系 + 创意点

Step 2: 选择年代
  └── 现代 / 80年代 / 历史 / 修仙 / 末世

Step 3: 确定身份+职业
  └── 身份决定爽点来源，职业决定金手指方向

Step 4: 选择关系组合
  └── 夫妻/闺蜜/婆媳等，决定核心冲突

Step 5: 确定金手指
  └── 能力型/身份型/辅助型

Step 6: 加入社会矛盾
  └── 增加真实感和讨论度

Step 7: 代入7阶段结构
  └── 输出完整大纲
```

# 四、种子案例演示

## 4.1 用户种子输入

```
【种子】
核心情节：女主被闺蜜抢走老公，重生后复仇
核心关系：女主 × 闺蜜（抢男人） × 老公（渣男）
创意点：老公是上门女婿，岳父岳母一直看不起女主
```

## 4.2 AI发散变体

### 变体A：原版（现代都市）

| 变量 | 选择 | 说明 |
|------|------|------|
| 年代 | 现代都市 | 默认选择 |
| 身份 | 豪门少奶奶（上门女婿） | 岳父岳母看不起 |
| 职业 | 全职太太 | 被说"靠男人养" |
| 关系 | 女主×闺蜜×渣男 | 经典三角 |
| 金手指 | 预知未来+神医空间 | 复仇+自医 |
| 社会矛盾 | 彩礼/上门女婿被看不起 | 增加讨论度 |

**核心冲突设计：**
```
开场：婚礼上被当众抛弃（发现老公和闺蜜在现场）
中期：发现老公是凤凰男+闺蜜是绿茶
高潮：女主身份揭示（原是隐藏千金/有神医身份）
结局：渣男净身出户 + 女主嫁给真爱（霸道总裁/战少）
```

# 五、创作检查清单

## 5.1 开场检查（决定留存率）

- [ ] 前3章有强冲突吗？（离婚/背叛/阴谋）
- [ ] 读者知道主角的"憋屈"吗？
- [ ] 有"为什么不解释"的悬念吗？
- [ ] 第一章出现的主要人物有几个？（≤3个最好）
- [ ] 阶段1是否有至少1个反转？
- [ ] 阶段1是否埋入至少1个伏笔？

## 5.2 反派设计检查（必须通过）

- [ ] 每个反派是否有明确动机？
- [ ] 每个反派是否有清晰目标？
- [ ] 每个反派是否有具体策略？（不是无脑坏）
- [ ] 反派等级是否与主角成长匹配？
- [ ] 主角胜利是否有智慧或能力支撑？（禁止降智获胜）

## 5.3 节奏检查（决定完读率）

- [ ] 每3章有一个小爽点吗？
- [ ] 每10章有一个大高潮吗？
- [ ] 主角在进步/变强吗？
- [ ] 反派有给主角制造麻烦吗？
- [ ] 每个阶段是否有标注的反转节点？

## 5.4 伏笔检查（全部回收）

- [ ] 阶段1埋入的伏笔是否在阶段2有呼应？
- [ ] 阶段3-4埋入的伏笔是否在阶段5-6有回收？
- [ ] 全部伏笔是否在阶段6结束前回收完毕？
- [ ] 是否有悬空伏笔？（禁止）

## 5.5 场景角色检查（非强行添加）

- [ ] 每个路人角色删除后是否影响剧情？
- [ ] 路人角色是否服务于明确的功能？（信息传递/冲突推动/打脸工具）
- [ ] 是否没有为加而加的冗余角色？

## 5.6 爽点密度检查

| 阶段 | 爽点数量 | 类型 |
|------|---------|------|
| 第1-5章 | 1-2个 | 小爽（主角忍耐/伏笔） |
| 第6-15章 | 3-5个 | 中爽（主角反击） |
| 第16-25章 | 5-8个 | 大爽（打脸开始） |
| 第26-35章 | 10+个 | 连续爽（复仇完成） |

## 5.7 结局检查

- [ ] 结局是HE吗？（番茄读者不喜欢Be）
- [ ] 恶人有恶报吗？
- [ ] 主角得到幸福了吗？
- [ ] 有"意犹未尽"的感觉吗？

# 六、快速创作流程

## 6.1 接到种子后的操作步骤

```
Step 1: 分析种子
  └── 提取：核心冲突 + 人物关系 + 创意点

Step 2: 选择变体方向
  └── 用户指定 OR AI推荐 OR 随机抽取

Step 3: 填写变量矩阵
  └── 年代/身份/职业/关系/金手指/社会矛盾

Step 4: 生成7阶段大纲
  └── 按固定结构填充每阶段核心事件
  └── 标注每阶段反转节点（每阶段至少1个）
  └── 设计反派三要素（动机+目标+策略）
  └── 规划伏笔埋入和回收位置

Step 5: 产出前3章试写
  └── 验证节奏和爽点密度
  └── 检查反派设计是否符合三要素
  └── 检查伏笔是否已埋入

Step 6: 用户反馈后继续全稿
  └── 阶段6必须通过"主角制胜逻辑"检查
```

## 6.2 标准字数分配

| 阶段 | 章节 | 字数 | 说明 |
|------|------|------|------|
| 开场冲突 | 1-3章 | 3000字 | 必须强冲突 |
| 重生/穿越 | 4-5章 | 2000字 | 快速过渡 |
| 小试牛刀 | 6-10章 | 5000字 | 第一次爽 |
| 中段积累 | 11-20章 | 10000字 | 持续爽 |
| 大高潮前奏 | 21-25章 | 5000字 | 决战前夕 |
| 大高潮 | 26-30章 | 5000字 | 连续爆爽 |
| HE结局 | 31-35章 | 5000字 | 完美收尾 |
| **总计** | **35章** | **35000字** | 最优区间 |
