## Step 1：搜索文章

### 规则：先查本地记录，再请求API

**记录目录：** `/root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/`

**流程：**
```
① 查询本地记录
   └─ ls /root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/
   └─ 检查是否有相关关键词的历史记录（7天内）

② 有记录（7天内）
   └─ 直接使用，从文件中读取文章列表
   └─ 标记 article.used = true（当篇文章使用后）

③ 无记录或过期
   └─ 请求 API
   └─ 保存结果到文件
   └─ 下次优先复用
```

### 命令示例

```bash
# 检查本地记录
ls /root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/健康*.json

# 无记录时请求API
cd /root/.openclaw/workspace/skills/wechat-search-redfox/scripts
REDFOX_API_KEY="***" python3 fetch_gzh_trends.py --keyword "健康" --start-date 2026-05-27 > result.json

# 保存到记录目录
cp result.json "/root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/健康-$(date +%Y-%m-%d).json"
```

### 记录格式

```json
{
  "keyword": "健康",
  "search_date": "2026-05-28",
  "articles": [
    {
      "title": "文章标题",
      "author": "作者",
      "url": "https://...",
      "clicksCount": 100001,
      "totalScore": 14.78,
      "summary": "摘要...",
      "publicTime": "2026-05-27",
      "used": false
    }
  ]
}
```

### 选择标准

- 评分 > 13.0（高分爆款）
- 阅读数 10w+（传播广）
- 主题符合目标受众（50岁退休老人）

## Step 3：改写标题+正文+摘要

### 改写前必须先给用户选择，用户确认后再执行！

#### 改写方案选项（每次提供2-3个供用户选择）

```
改写方案A：【保留原文结构】
- 保留原文整体框架和段落顺序
- 只调整表达方式（换词、换句式）
- 适当精简过渡句和重复表述
- 字数保留：90%以上

改写方案B：【换角度/换切入点】
- 换一个新的开头角度
- 保留原文核心内容
- 语言更贴近目标读者
- 字数保留：80-90%

改写方案C：【精简版】
- 保留核心观点和关键案例
- 大幅精简细节描述
- 适合快速阅读
- 字数保留：50-70%
```

#### 输出格式（改写前发给用户选择）

```markdown
## 改写方案选择

### 原文标题
《原标题》

### 原文摘要
（原文摘要内容）

### 改写方案A：保留原文结构
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：调整表达方式，保留原文框架

### 改写方案B：换角度
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：从XX角度切入，保留核心内容

### 改写方案C：精简版
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：保留核心观点，大幅精简细节

---请选择一个方案（A/B/C）：___
```

#### 改写规则

| 规则 | 说明 |
|------|------|
| **字数保留** | 改写后至少保留原文80%以上内容（除非用户选择精简版） |
| **核心内容** | 必须保留原文的核心观点、关键案例、数据 |
| **换表达** | 只换表达方式，不删减核心信息 |
| **目标读者** | 根据目标人群（50岁退休老人）调整语言风格 |
| **配图位置** | 根据内容段落匹配图片，提前规划配图位置 |

#### 阿呆操作

1. 阅读原文，理解核心内容和结构
2. 生成2-3个改写方案选项
3. 每个方案包含：新标题、新摘要、字数预估、改写简述
4. **发用户选择，等待确认后再执行改写**

## Step 5：搜索配图并下载

### 图片数量判断

| 原文字数 | 建议配图数量 |
|---------|-------------|
| < 2000字 | 2-3张 |
| 2000-5000字 | 3-5张 |
| 5000-10000字 | 5-8张 |
| > 10000字 | 8张以上 |

### 配图位置规划

**图文混排 = 文字-图片-文字 交替排列**
```
段落1文字（200-400字）
[图片1]
段落2文字（200-400字）
[图片2]
段落3文字（200-400字）
[图片3]
...
```

**位置选择原则：**
- 图片必须与当前章节/段落内容高度相关
- 根据章节主题选择对应图片（如：讲烧烤就用烧烤图，讲咖啡就用咖啡图）
- 开头第一张图：吸引读者
- 结尾避免单独放图
- 每张图片需确认与上下文的契合度

### 图片来源

| 来源 | 链接格式 |
|------|---------|
| Unsplash | `https://images.unsplash.com/photo-xxx?w=600` |
| Pexels | `https://images.pexels.com/photos/xxx?w=600` |

### 下载命令

```bash
mkdir -p /root/.openclaw/workspace/图片素材
cd /root/.openclaw/workspace/图片素材
proxychains4 curl -s --max-time 15 -L "图片URL" -o 图片1.jpg
sleep 2
```

### 图片识别（确认与章节内容契合度）

```bash
# 分析图片内容
image /root/.openclaw/workspace/图片素材/图片1.jpg
# 确认图片是否与对应章节内容相关
```

**图片选择规则：**
| 章节主题 | 图片类型 |
|---------|---------|
| 烧烤/致癌 | 烧烤场景图 |
| 火锅/太烫 | 火锅场景图 |
| 咖啡/提神 | 咖啡场景图 |
| 健康检查 | 医院/医生/检查场景图 |
| 饮食习惯 | 健康饮食图 |
| 数据/统计 | 图表/数据可视化图 |

**图片与章节对照表（上传前必须确认）：**
```
章节1标题 → 图片1（主题相关）
章节2标题 → 图片2（主题相关）
章节3标题 → 图片3（主题相关）
...
```

## Step 6：上传飞书

### 图文混排上传流程

**核心：文字-图片-文字 交替排列，不是图片堆在末尾**

#### 6.1 创建含占位符的文档

```bash
cd /root/.openclaw/workspace

# 文档内容中图片位置用 [图片N] 占位符标记
lark-cli docs +create \
  --api-version v2 \
  --title "【改写】文章标题" \
  --content "$(cat /root/.openclaw/workspace/改写素材/改写版_humanize.md)" \
  --doc-format markdown
```

**注意：** 图片位置在创建文档时就要规划好，不是创建完再追加

#### 6.2 在文档中插入图片（图文交替）

**每张图片插入到对应段落后面，保持图文交替排列**

```bash
cd /root/.openclaw/workspace

# 插入图片1（在第一个占位符位置）
lark-cli docs +media-insert \
  --doc <文档ID> \
  --file "图片素材/图片1.jpg" \
  --selection-with-ellipsis "[图片1]"
sleep 2

# 插入图片2（在第二个占位符位置）
lark-cli docs +media-insert \
  --doc <文档ID> \
  --file "图片素材/图片2.jpg" \
  --selection-with-ellipsis "[图片2]"
sleep 2

# 继续插入后续图片...
```

#### 6.3 上传图片到云空间（如需要）


```bash
cd /root/.openclaw/workspace/图片素材

lark-cli drive +upload --file 图片1.jpg
sleep 2
lark-cli drive +upload --file 图片2.jpg
```

## 关键限制

| 问题 | 说明 |
|------|------|
| 微信图片被拦截 | 云服务器IP访问微信图片返回400，需从公开图库下载 |
| 飞书API权限 | `application:self_manage` 缺失时报错，用 lark-cli 绕过 |
| 图片文字环绕 | 飞书文档API不支持图文混排 |
| RedFox API计费 | 搜索结果需记录到本地，减少重复请求 |
