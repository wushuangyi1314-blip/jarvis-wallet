## Step 1：搜索文章

### 规则：先查本地记录，再请求API

**记录目录：** `/root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/`

**流程：**
```
① 查询本地记录
   └─ ls /root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/
   └─ 检查是否有相关关键词的历史记录（7天内）

② 有记录（7天内）
   └─ 直接使用，从文件中读取文章列表
   └─ 标记 article.used = true（当篇文章使用后）

③ 无记录或过期
   └─ 请求 API
   └─ 保存结果到文件
   └─ 下次优先复用
```

### 命令示例

```bash
# 检查本地记录
ls /root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/健康*.json

# 无记录时请求API
cd /root/.openclaw/workspace/skills/wechat-search-redfox/scripts
REDFOX_API_KEY="***" python3 fetch_gzh_trends.py --keyword "健康" --start-date 2026-05-27 > result.json

# 保存到记录目录
cp result.json "/root/.openclaw/workspace/公众号文章改写/RedFox搜索记录/健康-$(date +%Y-%m-%d).json"
```

### 记录格式

```json
{
  "keyword": "健康",
  "search_date": "2026-05-28",
  "articles": [
    {
      "title": "文章标题",
      "author": "作者",
      "url": "https://...",
      "clicksCount": 100001,
      "totalScore": 14.78,
      "summary": "摘要...",
      "publicTime": "2026-05-27",
      "used": false
    }
  ]
}
```

### 选择标准

- 评分 > 13.0（高分爆款）
- 阅读数 10w+（传播广）
- 主题符合目标受众（50岁退休老人）

## Step 3：改写标题+正文+摘要

### 改写前必须先给用户选择，用户确认后再执行！

#### 改写方案选项（每次提供2-3个供用户选择）

```
改写方案A：【保留原文结构】
- 保留原文整体框架和段落顺序
- 只调整表达方式（换词、换句式）
- 适当精简过渡句和重复表述
- 字数保留：90%以上

改写方案B：【换角度/换切入点】
- 换一个新的开头角度
- 保留原文核心内容
- 语言更贴近目标读者
- 字数保留：80-90%

改写方案C：【精简版】
- 保留核心观点和关键案例
- 大幅精简细节描述
- 适合快速阅读
- 字数保留：50-70%
```

#### 输出格式（改写前发给用户选择）

```markdown
## 改写方案选择

### 原文标题
《原标题》

### 原文摘要
（原文摘要内容）

### 改写方案A：保留原文结构
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：调整表达方式，保留原文框架

### 改写方案B：换角度
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：从XX角度切入，保留核心内容

### 改写方案C：精简版
新标题：【新标题】
新摘要：【50字内摘要】
字数：约XXX字（保留XX%）
简述：保留核心观点，大幅精简细节

---请选择一个方案（A/B/C）：___
```

#### 改写规则

| 规则 | 说明 |
|------|------|
| **字数保留** | 改写后至少保留原文80%以上内容（除非用户选择精简版） |
| **核心内容** | 必须保留原文的核心观点、关键案例、数据 |
| **换表达** | 只换表达方式，不删减核心信息 |
| **目标读者** | 根据目标人群（50岁退休老人）调整语言风格 |
| **配图位置** | 根据内容段落匹配图片，提前规划配图位置 |

#### 阿呆操作

1. 阅读原文，理解核心内容和结构
2. 生成2-3个改写方案选项
3. 每个方案包含：新标题、新摘要、字数预估、改写简述
4. **发用户选择，等待确认后再执行改写**

## Step 5：AI生成配图

### 图片数量判断

| 原文字数 | 建议配图数量 |
|---------|-------------|
| < 2000字 | 2-3张 |
| 2000-5000字 | 3-5张 |
| 5000-10000字 | 5-8张 |
| > 10000字 | 8张以上 |

### 配图位置规划

**图文混排 = 文字-图片-文字 交替排列**
```
段落1文字（200-400字）
[图片1]
段落2文字（200-400字）
[图片2]
段落3文字（200-400字）
[图片3]
...
```

**位置选择原则：**
- 图片必须与当前章节/段落内容高度相关
- 根据章节主题选择对应图片（如：讲烧烤就用烧烤图，讲咖啡就用咖啡图）
- 开头第一张图：吸引读者
- 结尾避免单独放图
- 每张图片需确认与上下文的契合度

### 图片生成（AI生产）

**图片统一由大模型生成，不从网上搜索下载。**

**生成步骤：**

1. **分析章节主题** → 确定图片需求
2. **编写提示词** → 描述场景、风格、内容
3. **调用 image_generate** → 生成图片
4. **上传飞书云空间** → 获取链接

**提示词编写规则：**
- 简洁明确：描述主体、场景、风格
- 加入 "realistic photography style" 等风格描述
- 避免文字/LOGO
- 适合中国读者审美

**示例提示词：**
```
# 烧烤图
A steaming BBQ grill with sizzling meat skewers, charcoal fire, smoke rising, realistic food photography, warm colors, appetizing, Chinese street food style. No text.

# 火锅图
A steaming hot pot (huoguo) with red broth, tofu, vegetables, and meat. The pot is bubbling on a table with chopsticks and small bowls. Realistic food photography style, warm colors, appetizing, delicious Chinese cuisine. No text.

# 医生问诊图
A doctor in white coat with stethoscope, examining a patient in a modern clinic, realistic medical photography style, professional and trustworthy atmosphere. No text.
```

### 图片与章节对照表

| 章节主题 | 图片类型 | 提示词要点 |
|---------|---------|-----------|
| 烧烤/致癌 | 烧烤场景图 | 炭火烤肉、烟熏风格 |
| 火锅/太烫 | 火锅场景图 | 红油翻滚、蒸汽 |
| 咖啡/提神 | 咖啡场景图 | 咖啡杯、拉花 |
| 健康检查 | 医院/医生图 | 白大褂、诊室 |
| 遗传/基因 | DNA/实验室图 | 显微镜、细胞 |
| 预防胃癌 | 健康饮食图 | 蔬果、沙拉 |

**上传飞书云空间：**


**获取飞书链接：**
返回的 `url` 字段即为文档可用的图片链接。

## Step 6：上传飞书

### 图文混排上传流程

**核心：文字-图片-文字 交替排列，不是图片堆在末尾**

#### 6.1 创建含占位符的文档

```bash
cd /root/.openclaw/workspace

# 文档内容中图片位置用 [图片N] 占位符标记
lark-cli docs +create \
  --api-version v2 \
  --title "【改写】文章标题" \
  --content "$(cat /root/.openclaw/workspace/改写素材/改写版_humanize.md)" \
  --doc-format markdown
```

**注意：** 图片位置在创建文档时就要规划好，不是创建完再追加

#### 6.2 在文档中插入图片（图文交替定位插入）

**关键：每张图片用 `--selection-with-ellipsis "[图片N]"` 精确定位到占位符，自动 insert_after**

```bash
# 先cd到图片所在目录（lark-cli要求相对路径，不能用绝对路径）
cd /root/.openclaw/media/tool-image-generation

# 插入图片1到第一个占位符后面
lark-cli docs +media-insert \
  --doc <文档ID> \
  --file image-1.png \
  --selection-with-ellipsis "[图片1]" \
  --align center
sleep 3

# 插入图片2到第二个占位符后面
lark-cli docs +media-insert \
  --doc <文档ID> \
  --file image-2.png \
  --selection-with-ellipsis "[图片2]" \
  --align center
sleep 3

# 以此类推...
```

**⚠️ 常见失败原因：**
- `--file` 必须是当前目录下的相对路径，绝对路径报 `unsafe file path`
- 每次插入间隔至少3秒，避免API响应不稳
- 占位符在文档中必须唯一（重复会出现定位失败）
- 插入后占位符消失是正常现象（被当作定位锚点消费掉了）
- 插入失败报 `API returned empty JSON` 时，重试一次即可

#### 6.3 上传图片到云空间（如需要）


```bash
cd /root/.openclaw/workspace/图片素材

lark-cli drive +upload --file 图片1.jpg
sleep 2
lark-cli drive +upload --file 图片2.jpg
```

## 附录：AI配图生成强相关规则（2026-05-29 新增）

### 核心理念

**图片是文章段落的"视觉翻译"——段落讲什么，图就画什么。**

段落提到"心脏"就画心脏，提到"跑步的年轻人"就画跑步的人。不能泛化、不能抽象、不能偷懒。

### 强相关四步检验法

```
Step 1: 读段落 → 提取物理实体（什么物体/人/场景在段落里被描述）
Step 2: 看图片 → 图片的主体是那个物理实体吗？
Step 3: 想关联 → 读者看到这张图，能联想到段落的关键词吗？
Step 4: 不通过 → 重写prompt，重新生成
```

### prompt编写核心公式

```
[物理主体] + [场景/状态] + [风格/氛围] + [禁止项]
```

| 要素 | 说明 | 示例 |
|------|------|------|
| 物理主体 | 段落里出现的东西 |烤鸭、心脏、跑步的人、咖啡杯|
| 场景/状态 | 主体在做什么/什么样 | 烤鸭在盘子里、心脏有斑块、跑步者满头大汗 |
| 风格/氛围 | 医学用clinical，生活用lifestyle | realistic photography, clinical atmosphere |
| 禁止项 | 末尾统一加 | No text, no labels, no logos. |

### 常见错误与正确做法

| 错误 | 原因 | 正确 |
|------|------|------|
| 生成"健康生活"抽象图 | 物理主体不具体 | 画一个跑步的人 |
| 生成"风险上升"图表 | 公众号不需要图表 | 画一个超重的人站在秤上 |
| 讲烤鸭用火锅图 | 主体与内容不符 | 画一盘烤鸭 |
| 讲心脏用抽象图 | 医学内容需要真实感 | 画心脏解剖示意 |

### 配图生成禁止项

1. **禁止图表/数据图** —— 公众号不需要看图表
2. **禁止抽象概念图** —— "健康""风险""幸福"无法画成图
3. **禁止无关替代** —— 不能因为不会画烤鸭就随便用个食物图凑数
4. **禁止泛化提示词** —— `"a healthy lifestyle"` 这类prompt无法生成强相关图片

### 示例：强相关prompt生成过程

**段落原文：**
> 20多岁就肥胖的人，其心血管状态往往在35岁时就已经相当于健康人群45岁的水平。

**提取物理主体：** 心脏、血管

**生成prompt：**
```
A human heart with visible artery blockage, realistic medical illustration, clinical atmosphere, red and orange warning colors indicating strain. No text, no labels.
```

**段落原文：**
> 一个简单自测：过去3年，你的体重是不是每年增加超过1.5公斤？

**提取物理主体：** 体重秤、超重的人

**生成prompt：**
```
A person standing on a bathroom scale showing a significant weight increase, realistic home environment, concerned expression, clean photography. No text, no labels.
```

### 图片数量规划

基于改写后字数决定：
- < 2000字 → 2-3张
- 2000-5000字 → 3-5张
- 5000-10000字 → 5-8张
- > 10000字 → 8张以上

### 位置选择原则

- 首图：放第一个数据段之后，吸引读者
- 中间图：每个核心观点配一张，图文交替
- 结尾图：避免单独放图，放在总结段落后
