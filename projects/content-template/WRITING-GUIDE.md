# 运营写作指南

> **核心原则：写读者真正想看的东西，不是写给搜索引擎看的东西。**

---

## 一、写作前：先做研究

在开始写正文之前，先完成以下研究。这不是浪费时间，而是确保文章有"观点"而不是泛泛而谈。

### 必做研究清单

1. **这个工具到底是什么？**
   - 官方定位是什么？（不要自己猜）
   - 核心功能是什么？（列出 3 个最重要的）
   - 定价模式是什么？（免费/订阅/一次性？）

2. **这个工具和竞品相比，最大的差异点是什么？**
   - 用户为什么选它而不是竞品？
   - 它最常被拿来和谁对比？（搜索这个工具名称时，相关搜索会告诉你）

3. **这个工具最大的 1-2 个优点和 1-2 个缺点是什么？**
   - 要有判断，不要两边都夸
   - 如果优点和缺点都说不出来，说明研究得还不够深

4. **这个工具适合谁、不适合谁？**
   - 一句话描述目标用户
   - 什么情况下读者应该选它，什么情况下应该选竞品

5. **写一个一句话总结**
   - 类似于"如果你想要 X，选它；如果你更看重 Y，选竞品"
   - 这个总结会成为文章的核心观点

---

## 二、写作结构：自然展开就好

不需要按照固定的"标题模板"来写。文章的结构应该由内容本身决定。

### 建议的写作顺序

```
1. 开头（200字以内）
   - 直接告诉读者这篇文章解决什么问题
   - 可以用"我为什么写这篇文章"的角度切入

2. 核心体验（这是最重要的部分）
   - 你真正用这个工具之后，发现的"用了才知道"的细节
   - 不要只列功能，要写感受和判断
   - 要有具体例子，不要泛泛而谈

3. 适用场景分析
   - 这个工具适合谁
   - 这个工具不适合谁
   - 和竞品相比，什么时候选它、什么时候不选它

4. 工具对比（如果有）
   - 用表格对比核心参数（价格、功能、适用场景）
   - 不要只放数据，要写"我的建议是什么"

5. 总结
   - 一句话核心观点
   - 读者下一步怎么做
```

### 常见内容元素及写法

| 元素 | 写法 | 注意事项 |
|---|---|---|
| 工具对比 | Markdown 表格 `\| 工具 \| 价格 \| 功能 \|` | 表格标题行要清晰 |
| 优缺点 | 可以用表格，也可以用 `✅ / ❌` 列表 | 要有判断，不要只罗列 |
| 引用块 | `>` 开头 | 用于放"一句话总结"或"关键观点" |
| 代码块 | \`\`\`语言名称 | 只在技术教程类文章中使用 |
| 列表 | `1.` 或 `-` 都可以 | 同篇文章保持一致 |

## 三、避免套路化的方法

套路化的文章通常有这些特征：
- 每个工具都是"功能强大、界面友好、性价比高"
- 优缺点永远是"优点：功能多；缺点：学习曲线陡"
- 结尾永远是"总体来说是一款优秀的工具"

### 真正有价值的写作

**套路化的写法：**
> ChatGPT 的优点是功能强大，可以用于写作、编程、分析等多种场景。缺点是免费版有使用限制。

**有判断的写法：**
> ChatGPT 的强项是**对话式的上下文理解**——你能用它做多轮头脑风暴，中间打断它说"不对，我换个方向"，它真的能跟上。但如果你的场景是"给我一个确定的答案然后我不用再改"，ChatGPT 反而不如专门的工具来得干脆。

区别在于：第一个写法是在"描述"，第二个写法是在"判断"。读者需要的是判断。

## 四、frontmatter 填写规范

frontmatter 是文章的元数据，只需要填写以下字段：

```yaml
---
title: "文章标题"
description: "一段话描述文章内容，会显示在搜索结果中"
date: 2026-04-01
heroImage: "xxx.jpg"
rating: 4.8
featuredTool: "工具名称"
---
```

**其他字段都是可选的**，不需要你填。RD Agent 会根据文章内容自动处理。

### 字段说明

| 字段 | 是否必填 | 说明 |
|---|---|---|
| title | ✅ | 文章标题 |
| description | ✅ | 150 字以内的描述 |
| date | ✅ | 发布日期，格式 `YYYY-MM-DD` |
| heroImage | ✅ | 头图文件名，放在 `static/assets/images/` 目录 |
| rating | 推荐 | 评分，1-5 分 |
| featuredTool | 推荐 | 文章主要讨论的工具名称 |

---

## 五、检查清单

写完文章后，自己检查一下：

- [ ] 文章有我自己的核心观点吗？还是只是在描述功能？
- [ ] 有一两个"用了才知道"的细节吗？
- [ ] 有没有写这个工具"不适合谁"？
- [ ] 竞品对比有具体数据，不是泛泛而谈吗？
- [ ] 结尾的一句话总结能让人记住吗？

如果这 5 个问题都能回答"是"，文章质量基本过关。
