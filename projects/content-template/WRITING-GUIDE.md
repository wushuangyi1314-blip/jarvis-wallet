# 运营写作指南

> 版本：2.0 | 更新：2026-04-17
> 用途：指导 AI 工具评测文章的写作标准

---

## 一、写作前研究清单

完成文章前，必须完成以下研究：

### 1.1 工具研究

- [ ] 这个工具能做什么？核心功能是什么？
- [ ] 目标用户是谁？
- [ ] 定价模式是什么？（免费/订阅/一次性）
- [ ] 相比竞品有什么独特优势？

### 1.2 竞品对比

- [ ] 同类工具里它排第几？
- [ ] 竞品 A/B/C 分别是什么？
- [ ] 各自优缺点是什么？

### 1.3 用户视角

- [ ] 适合什么人用？
- [ ] 有什么使用门槛？
- [ ] 用户真实评价如何？

---

## 二、文章结构

### 2.1 推荐结构（自然展开）

```
1. 开篇：场景引入（不用模板）
2. 概述：工具是什么，能解决什么问题
3. 核心功能：2-3个重点功能详细介绍
4. 优缺点：客观真实
5. 定价：清晰透明
6. 适用场景：适合谁/不适合谁
7. 总结：给出建议
```

### 2.2 禁止模板

- 不要用"随着AI技术的快速发展..."
- 不要用"今天给大家介绍一款..."
- 不要堆砌功能列表

---

## 三、质量标准

### 3.1 内容质量

- [ ] 有独立观点，不是罗列功能
- [ ] 案例具体，不是空泛描述
- [ ] 优缺点客观真实
- [ ] 有实际使用场景

### 3.2 禁止事项

- ❌ 堆砌功能列表（水字数）
- ❌ 无脑吹（太软没有可信度）
- ❌ 纯负面（没有参考价值）
- ❌ 抄袭官方介绍（要有自己的话）

### 3.3 案例要求

每个功能介绍最好包含：
- 具体使用场景
- 实际效果描述
- 用户反馈引用（如有）

---

## 四、Frontmatter 规范

### 4.1 必须字段

```yaml
title: "工具名 + Review/测评/年"
date: "2026-04-17"
description: "一句话描述这个工具能解决什么问题"
categories: ["AI工具类型"]
tags: ["工具名", "相关标签1", "相关标签2"]
```

### 4.2 必须包含 TOC

```yaml
toc:
  - id: "overview"
    title: "概述"
  - id: "features"
    title: "核心功能"
  - id: "pricing"
    title: "定价"
  - id: "pros-cons"
    title: "优缺点"
  - id: "conclusion"
    title: "总结"
```

---

## 五、写后自检

| 检查项 | 要求 |
|-------|------|
| 标题 | 吸引人但不过度标题党 |
| 开篇 | 场景引入自然，不套模板 |
| 内容 | 有观点有案例，不是功能堆砌 |
| 优缺点 | 客观真实，有具体说明 |
| 结尾 | 有明确建议 |
| 字数 | 1500-3000字 |
| 错别字 | 无明显错别字/病句 |
| Frontmatter | 完整且正确 |
