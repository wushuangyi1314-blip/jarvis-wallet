# 文章自检清单

> 写完文章后，用这个清单自我检查。

---

## 必检项

- [ ] **有核心观点吗？** 
  - 不是在描述功能，而是在表达"我认为这个工具怎么样"
  - 能用一句话说清楚文章的核心观点吗？

- [ ] **有"用了才知道"的细节吗？**
  - 至少有一两个只有真正用过才能写出来的观察
  - 不是从官网抄的功能列表

- [ ] **写了"不适合谁"吗？**
  - 没有工具是万能的
  - 告诉读者什么时候应该选竞品

- [ ] **表格数据具体吗？**
  - 价格是真实的具体数字，不是"价格实惠"
  - 对比有具体场景，不是罗列功能点

- [ ] **结尾让人记住什么？**
  - 有一句话总结
  - 读者知道下一步该做什么

---

## 格式检查

- [ ] frontmatter 的 `title`、`description`、`date` 填好了
- [ ] `heroImage` 对应的图片文件已上传到 `static/assets/images/`
- [ ] 没有用 HTML 标签（如 `<table>`、`<div>`），全部用 Markdown
- [ ] 表格用了 `|` 格式，有 `|---|` 分隔符行
- [ ] 标题层级是 `## H2` → `### H3`，没有跳级

---

## 完成后

1. 运行检查脚本：`./projects/scripts/pre-commit-check.sh <文章路径>`
2. 提交到 GitHub
3. 等 CI/CD 自动构建并部署
4. 检查线上页面样式是否符合预期
