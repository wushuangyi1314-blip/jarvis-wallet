# 写后检查清单

> 版本：1.0 | 日期：2026-04-01
> 用途：运营写完文章后自检，确保质量再提交

---

## 一、内容自检

| 检查项 | 要求 | 状态 |
|-------|------|:----:|
| 有独立观点 | 不是堆砌功能，有自己的评价 | ☐ |
| 案例具体 | 有实际使用场景，不是空泛描述 | ☐ |
| 优缺点客观 | 不无脑吹，不刻意贬低 | ☐ |
| 无废话段落 | 每段都有实质内容 | ☐ |
| 字数达标 | 1500-3000字 | ☐ |

---

## 二、标题检查

| 检查项 | 要求 | 状态 |
|-------|------|:----:|
| 吸引人 | 能引起目标用户兴趣 | ☐ |
| 不过度标题党 | 避免"震惊！"等极端词汇 | ☐ |
| 包含关键词 | 工具名 + 核心价值 | ☐ |

---

## 三、Frontmatter 检查

| 检查项 | 要求 | 状态 |
|-------|------|:----:|
| title | 工具名 + Review/测评 | ☐ |
| date | 格式正确，如 2026-04-17 | ☐ |
| description | 一句话描述，不超过150字 | ☐ |
| categories | 正确分类，如 AI Coding | ☐ |
| tags | 包含工具名和相关标签 | ☐ |
| toc | 必须包含，且 id 唯一 | ☐ |

---

## 四、格式检查

| 检查项 | 要求 | 状态 |
|-------|------|:----:|
| 无错别字 | 通读一遍无明显错误 | ☐ |
| 无病句 | 句子通顺，无语法错误 | ☐ |
| 标题层级 | H2/H3 使用合理 | ☐ |
| 列表格式 | 无序列表用 - | ☐ |

---

## 五、发布前确认

| 检查项 | 状态 |
|-------|:----:|
| 文章内容完整 | ☐ |
| Frontmatter 完整 | ☐ |
| TOC 链接正确 | ☐ |
| 图片已上传到正确位置 | ☐ |
| Hugo 构建成功 | ☐ |
