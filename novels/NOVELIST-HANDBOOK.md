## 二、Agent 创作执行流程（5阶段）

### 阶段0 - 接任务

**确认信息：**
- 写哪几章
- 哪个品类
- 当前卷的核心事件
- 上一章结尾的钩子
- 字数要求（默认2000-3000字）

---

### 阶段1 - 规划阶段

**必须读取：**
- `/workspace/novels/projects/<项目名>/PLAN.md`（对应项目规划书）
- 前3章剧情（新卷还要读卷纲）

**规划书查表：**
- 本章情节属于规划书哪个章节？→ 对应套路是什么？
- 本章人物是谁？→ 对应人物小传的设定
- 本章场景在哪？→ 对应地图/世界设定

**提取信息：**
- 主角当前目标
- 本章要推进的情节
- 本章爽点（如果有）
- 结尾钩子
- 文笔风格

**产出：** 本章规划单

---

### 阶段2 - 创作阶段

**【引用规划单检查】**
- 已确认本章核心事件：_______
- 已确认本章爽点：_______
- 已确认结尾钩子：_______
- 本章情节对应规划书：_______
- 本章人物对应规划书人物：_______

**开篇（前200字）**
公式：冲突/危机 + 主角行动 + 悬念

开篇十忌：
- 大段环境描写
- 主角内心独白
- 背景介绍
- 陈词老套
- 故弄玄虚

**中段（发展）**
公式：目标 → 障碍 → 行动 → 结果

**高潮（爆发点）**
公式：压抑 → 爆发 → 结果

**结尾（断章）**
公式：情绪最高点 + 戛然而止

类型：
- A. 悬念式
- B. 冲突式
- C. 反转式
- D. 危机式

---

### 阶段3 - 自检阶段

**必检项：**
- 主角行为是否符合人设
- 所有人物行为符合逻辑
- 有冲突/悬念
- 无废话/水段落
- 结尾钩子够强
- 字数2000-3000

**套路化检查：**
- 这个情节在其他书里见过吗
- 独特记忆点是什么
- 有没有刻意设计感
- 主角有做决定/行动吗
- 这章删掉对故事有影响吗

**爽点检查（如有）：**
- 积累够吗
- 爆发够爽吗
- 引爆方式是什么
- 有旁观者吗

---

### 阶段4 - 定稿阶段

**通读检查：**
- 错字/病句
- 对话生动
- 场景具体
- 情绪展示

**去AI味处理（必须执行）：**
使用 `humanize-chinese-2-0-0` skill 进行处理：
- 删除总结性语句
- 打碎工整句式
- 加口癖/口头禅
- 词汇口语化

**最终检查：**
- 必检项全部通过
- 套路化检查通过
- 结尾钩子够强
- AI评分 < 50

## 三、写作技巧速查

### 3.1 人物刻画

**核心原则：** 行为体现性格，不靠描写堆砌

**检查清单：**
- [ ] 主角每个选择都符合人设
- [ ] 配角有自己的动机
- [ ] 反派行为有合理逻辑
- [ ] 没有工具人（每个角色推动剧情外还有自己的故事）

### 3.2 逻辑严谨

**战斗逻辑：**
- 实力差距 > 3倍，基本无法靠智谋弥补
- 地形/道具/环境可以创造以弱胜强的机会
- 金手指必须有明确代价

**伏笔埋设：**
- 每个伏笔在埋下后3-5章内必须有呼应
- 长期伏笔需要在关键节点提醒读者

### 3.3 爽点设计

**爽感公式：**
```
爽感 = (铺垫时长 × 细节密度) / (预期偏差度 + 1)
```

**常见爽点类型：**
- 打脸（压抑越久，爆发越爽）
- 逆袭（以弱胜强，需要足够铺垫）
- 收获（得到宝物/能力/地位）
- 复仇（因果报应，节奏要快）

## 四、常见错误清单

### 4.1 必踩的坑

1. **开篇堆设定** - 上来就介绍世界观，读者跑了
2. **主角太弱还浪** - 读者会弃书
3. **反派智商掉线** - 爽感全无
4. **升级太容易** - 没有成就感
5. **感情戏尴尬** - 心理描写不够就用动作/对话补

### 4.2 弃书节点

| 节点 | 原因 |
|------|------|
| 前3章 | 开篇不抓人 |
| 第10章 | 没有爽点 |
| 第30章 | 套路重复 |
| 第100章 | 节奏拖沓 |
