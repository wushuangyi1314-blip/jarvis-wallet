# 小说工厂工作流 v3.0

> 版本：3.0 | 日期：2026-04-16

---

## 一、整体架构

```
用户/阿呆提供创意
        ↓
① 创意立项 → 选定大分类（玄幻/都市/网游/末日/历史/科幻）
        ↓
② 风格确认 → 套路流派 + 文笔风格
        ↓
③ 世界观规划 → 输出《小说规划书》
        ↓
④ 小说创作 → 按套路模板 + 写作手册
        ↓
⑤ 质量把控 → 阿呆审读 → 修改 → 定稿
```

---

## 二、角色分工

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 用户（高总） | 提供创意、确认规划书 | 创意需求 | 确认各阶段产出 |
| 阿呆（项目经理） | 协调、审读、质控 | 用户需求 | 修改意见、定稿 |
| 小说家Agent | 按规划书执行创作 | 规划书+手册 | 章节初稿 |

## 四、工作流详解

### 第一步：创意立项

输入：用户的一句话创意
输出：确定大分类

### 第二步：风格确认

输出：套路流派 + 文笔风格

文笔风格：简洁干练 / 细腻厚重 / 轻松幽默 / 严肃文学风

### 第三步：世界观规划

输出：《小说规划书》
保存到：/workspace/novels/projects/<项目名>/PLAN.md

---

### 第四步：小说创作（Agent执行流程）

**Agent必须读取的文件：**

**Agent执行流程（5阶段）：**

---

**阶段0：接任务** 【来自写作手册v4.0】

确认：
- 写哪几章 / 哪个品类
- 当前卷的核心事件
- 上一章结尾的钩子
- 字数要求（默认2000-3000）

---

**阶段1：规划阶段** 【来自写作手册v4.0】

必须读取：
- /workspace/novels/projects/<项目名>/PLAN.md
- 前3章剧情（新卷还要读卷纲）

**规划书查表：**
- 本章情节属于规划书哪个章节？→ 对应套路是什么？
- 本章人物是谁？→ 对应人物小传的设定
- 本章场景在哪？→ 对应地图/世界设定

提取信息：
- 主角当前目标
- 本章要推进的情节
- 本章爽点
- 结尾钩子
- 文笔风格

产出：本章规划单

---

**阶段2：创作阶段** 【来自写作手册v4.0】

**【引用规划单检查】**
- 已确认本章核心事件：_______
- 已确认本章爽点：_______
- 已确认结尾钩子：_______
- 本章情节对应规划书：_______
- 本章人物对应规划书人物：_______

**开篇（前200字）**
公式：冲突/危机 + 主角行动 + 悬念

开篇十忌：大段环境描写 / 主角内心独白 / 背景介绍 / 陈词老套 / 故弄玄虚

**中段（发展）**
公式：目标 → 障碍 → 行动 → 结果

**高潮（爆发点）**
公式：压抑 → 爆发 → 结果

**结尾（断章）**
公式：情绪最高点 + 戛然而止
类型：悬念式 / 冲突式 / 反转式 / 危机式

---

**阶段3：自检阶段** 【来自写作手册v4.0】

必检项：人设符合 / 逻辑无误 / 有冲突悬念 / 无水段落 / 钩子够强 / 字数2000-3000

套路化检查：情节见过吗 / 独特记忆点 / 刻意设计感 / 主角有行动 / 删掉有影响吗

爽点检查（如有）：积累够吗 / 爆发够爽吗 / 引爆方式 / 有旁观者吗

---

**阶段4：定稿阶段** 【来自写作手册v4.0 + humanize-chinese-2-0-0】

通读检查：错字病句 / 对话生动 / 场景具体 / 情绪展示

**去AI味处理：**

最终检查：必检项 / 套路化 / 结尾钩子 / AI评分<50

---

### 第五步：质量把控

阿呆审读初稿 → 提修改意见 → Agent修改 → 定稿 → 交付用户
