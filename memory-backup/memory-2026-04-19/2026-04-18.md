### 规则补充（已写入 MEMORY.md）

**文件版本判断规则：**
- 讨论文件版本前，先 git fetch origin + 比较 HEAD vs origin/main
- 用 git show origin/main:<file> 获取权威版本
- 说"旧版/新版"前必须有 git log/diff 实际数据支撑
- 不确定时先诊断再下结论，不凭记忆推断

**Git 分支操作规则：**
- 用临时分支开发时，每次 push 后立即同步 local main
- 删除临时分支前，必须确认 local main 与 origin/main 同步
- 任何分支切换后，第一时间检查 git status 和 git log
- 合并完成后立即删除临时分支，避免混淆
