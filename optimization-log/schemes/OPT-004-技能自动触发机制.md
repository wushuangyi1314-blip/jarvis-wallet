# OPT-004-技能自动触发机制

> 创建时间：2026-04-15
> 迭代时间：2026-04-17
> 状态：已完成

---

## 一、方案背景

用户希望技能能够根据上下文自动加载，而不是每次都需要手动触发或关键词匹配。实现语义触发（意图判断）方案。

---

## 二、核心方案：语义触发

**设计思想：不是关键词匹配，而是贝吉塔主动理解任务意图后决策**

```
用户下达任务
    ↓
贝吉塔理解任务本质
    ↓
判断是否需要调用技能 + 调用哪个
    ↓
加载技能并执行
```

---

## 三、自动触发配置

### Context Engineering Auto-Triggers（6个场景）

| 触发条件 | 自动读取 |
|----------|----------|
| 上下文压缩前 / ~80K tokens | context-compression |
| 分配2+ SubAgent | multi-agent-patterns |
| 任务失败重试3+次 | context-degradation |
| 构建工具/MCP | tool-design |
| 设置记忆系统 | memory-systems |
| 读取5+文件 | filesystem-context |

### Skill Auto-Triggers（5个场景）

| 触发条件 | 自动读取 |
|----------|----------|
| 用户说"找技能/安装技能" | find-skills |
| 用户说"整理记忆" | memory-tiering |
| 用户问"前端/网页设计" | frontend-design-3 |
| 用户问"记忆/知识图谱" | jpeng-knowledge-graph-memory |
| 用户问"任务/待办" | agent-task-tracker |

## 四、技能语义触发对照表

| 任务类型 | 判断标准 | 应触发技能 |
|---------|---------|-----------|
| 代码开发 | 需要写代码/调试/重构 | frontend-design-3 |
| Git操作 | 需要rebase/merge/解决冲突 | git-workflows |
| 容器部署 | 需要docker/镜像/容器管理 | docker-essentials |
| 数据库 | 需要设计表/写SQL/优化查询 | database-design |
| 浏览器自动化 | 需要操作浏览器/抓取数据 | playwright |
| 文案写作 | 需要写营销文案/广告语 | copywriting |
| 文本优化 | 需要去除AI痕迹/更自然 | humanize |
| UI/UX设计 | 需要设计界面/交互流程 | ui-ux-pro-max |
| 记忆整理 | 需要整理/归档/层级化记忆 | memory-tiering |
| 知识管理 | 需要构建/查询知识图谱 | jpeng-knowledge-graph-memory |
| 任务追踪 | 需要记录/跟踪/管理任务 | agent-task-tracker |
| 上下文优化 | 需要压缩/优化/管理上下文 | agent-skills-context-engineering |
| 技能查询 | 需要找/安装/管理技能 | find-skills |

---

## 五、禁止误触发规则

- 只有真正需要技能时才触发
- 用户只是提到某个词不等于需要该技能
- 根据任务本质判断，不是关键词匹配

---

## 六、效果评估

- ✅ 常见场景自动加载
- ✅ 复杂任务由贝吉塔协调分配
- ✅ 语义触发优于关键词匹配
