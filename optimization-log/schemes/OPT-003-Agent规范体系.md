# OPT-003-Agent规范体系

> 创建时间：2026-04-16
> 状态：已完成

---

## 一、方案背景

用户要求建立完整的Agent规范体系，覆盖所有角色，并建立保证AI按约定调用SubAgent的规范机制。

---

## 二、Agent角色定义

| Agent | 角色 | 职责 |
|-------|------|------|
| 阿呆 | 项目经理 | 接收需求、拆解任务、分配任务、汇总结果 |
| 研发 | 全栈工程师 | 代码开发、调试、部署 |
| 运营经理 | 内容运营 | 文案写作、营销策划 |
| 产品经理 | 产品规划 | 需求分析、PRD产出 |
| UI设计师 | 界面设计 | UI/UX设计 |
| 数据分析师 | 数据分析 | 数据提取、分析报告 |
| QA | 测试验证 | 测试验证、样式检查 |

---

## 三、回复标记规范

所有Agent回复末尾必须加Agent标记：

| Agent | 标记格式 |
|-------|----------|
| 阿呆（项目经理） | `——阿呆` |
| 研发 | `——研发` |
| 运营经理 | `——运营经理` |
| 产品经理 | `——产品经理` |
| UI设计师 | `——UI设计师` |
| 数据分析师 | `——数据分析师` |
| QA | `——QA` |

---

## 四、任务分发映射表

| 任务类型 | 判断标准 | 分配给 |
|---------|---------|--------|
| 代码/部署/技术开发 | 需要写代码/调试/部署 | 研发 |
| 内容写作、研究 | 需要写文案/营销内容 | 运营经理 |
| 产品规划、需求文档 | 需要分析需求/写PRD | 产品经理 |
| 界面设计 | 需要设计UI/交互 | UI设计师 |
| 数据分析 | 需要分析数据/提取 | 数据分析师 |
| 测试验证 | 需要验证功能/样式 | QA |
| 流程协调、跨Agent沟通 | 需要协调多Agent | 阿呆 |

---

## 五、SubAgent汇报规则

1. SubAgent完成后必须发微信给用户（带小尾巴）
2. 阿呆立即汇总转发
3. 禁止NO_REPLY
4. 禁止只说"完成"

---

## 六、效果评估

- ✅ 7个角色规范完整建立
- ✅ 任务分发清晰
- ✅ 汇报机制完善
