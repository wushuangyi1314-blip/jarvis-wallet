## 一、方案背景

用户对数字资产（技能、文档、知识库）要求完整可见性，不容忍"大概齐"的统计。技能操作必须获得明确授权。

---

## 二、核心规则

### 1. 安装前授权

**【硬性规则】技能操作必须获得明确授权**
- 未经用户明确授权，AI不得安装或删除任何技能
- 所有技能操作前必须与用户确认
- 获得同意后方可执行

### 2. 安装后同步

添加新技能时**必须同步更新任务分发规则**：
- 该技能负责处理哪些任务类型/场景
- 任务分发规则的维护是技能添加操作的必要组成部分
- 两者必须同时完成，缺一不可

### 3. 删除前备份

删除技能前必须commit空记录到Git，保留删除历史。

---

## 三、技能清单管理

### 安装记录格式

| 字段 | 说明 |
|------|------|
| 技能名称 | 技能的唯一标识 |
| 安装时间 | YYYY-MM-DD |
| Git Commit | push后的commit hash |
| 用途 | 这个技能解决什么问题 |
| 触发场景 | 什么情况下调用这个技能 |

### 同步要求

每次安装后必须同步到：
1. Git仓库（workspace）
2. Wiki知识库
3. unified/记忆
