# OPT-002-记忆系统与知识库分类方案

> 创建时间：2026-04-15
> 状态：已完成

---

## 一、方案背景

需要明确什么信息归属于记忆系统，什么信息归属于知识库。从信息来源到沉淀，建立完整的信息分类体系。

---

## 二、核心分类

### 记忆系统（Memory System）

**定位**：Agent的工作记忆，支持日常任务执行

**信息来源**：
- 对话记录（tdai自动提取）
- 任务执行过程
- 用户偏好和行为
- 场景上下文
- 每日工作记录

**沉淀位置**：
```
memory/unified/
├── contexts/     # 场景上下文
├── events/       # 对话事件记录
├── facts/        # 事实性信息
├── decisions/    # 决策记录
├── preferences/  # 用户偏好
└── tiering/      # 层级管理（hot/warm/cold）
```

### 知识库（Knowledge Base / Wiki）

**定位**：持久化知识，供AI查询推理

**信息来源**：
- 规则流程文档
- 项目技术决策
- 运营SOP
- 最佳实践
- 问题解决方案

**沉淀位置**：
```
wiki/main/
├── concepts/     # 概念定义
├── entities/     # 实体
├── syntheses/    # 综合知识
├── rules/        # 规则文档
├── processes/   # 流程文档
└── projects/    # 项目文档
```

---

## 三、信息流转

```
对话记录 → 记忆系统(自动+半自动) → 提炼 → 知识库(主动创建)
```

| 信息类型 | 沉淀方式 | 目标位置 |
|----------|----------|----------|
| 对话场景 | tdai自动提取 | unified/contexts/ |
| 对话事件 | tdai自动记录 | unified/events/ |
| 用户偏好 | AI分析提取 | unified/preferences/ |
| 重要决策 | AI提炼存入 | unified/decisions/ |
| 规则流程 | 主动创建 | wiki/rules/ |
| 技术方案 | 主动创建 | wiki/projects/ |

---

## 四、核心原则

> 记忆系统是"仓库"，知识库是"货架"
> - 仓库：进什么存什么（自动）
> - 货架：按分类摆好（主动）

---

## 五、效果评估

- ✅ 记忆系统与知识库职责清晰
- ✅ 信息流转路径明确
- ✅ 便于AI实时调用和推理
