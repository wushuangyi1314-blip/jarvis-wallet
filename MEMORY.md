## 文档发送规则（强制执行）

**文档发送规则（强制执行）**

**后续所有需要发给高总的文档，必须：**
1. 将原始文档内容转换为飞书文档
2. 发给高总飞书链接而非云盘/文件链接

**原因：** 云盘链接有访问限制，飞书文档可直接阅读无限制。

**文档版本迭代规则（强制执行）**

**每次文档版本迭代，必须：**
1. 创建全新飞书文档（新链接）
2. 新文档内容复制自老文档
3. 在新文档基础上做修改和调整
4. 修改文档标题的版本号（如v2.0 → v2.1）

**示例：**
- PRD v2.0（老文档）→ 复制内容到新文档 → 修改为v2.1 → 发新链接
- 不在老文档上直接修改
- 每个版本都是完整快照

**原因：** 避免block结构混乱、保留版本历史

## 高森项目团队 - 工作流程规范

### PM工作流程（强制执行，方案C）

```
高总提需求 → PM产出详细需求文档 → PM出关键页面线框草图（不精美） → 高总确认方向 → UI完整设计 → 高总确认设计稿
                     ↑必须停下确认                        ↑必须停下确认
```

| 阶段 | 产出物 | 要求 | 高总动作 |
|------|--------|------|----------|
| 1 | 高总提需求 | 文字描述 + 参考截图 | 发给PM |
| 2 | PM详细需求文档 | 功能点/布局/交互说清楚 | 确认"内容对不对" |
| 3 | 关键页面线框草图 | 2-3个关键页面，简单草图不用精美 | 确认"方向对不对" |
| 4 | UI完整设计稿 | 基于确认的方向出高保真 | 确认"设计对不对" |

**原则：线框原型不追求精美，只求方向正确。**

## PM产品需求文档编写规则（强制执行）

**PRD文档内容边界：**
- ✅ 可以写：功能描述、交互逻辑、页面结构、信息架构、用户体验流程、视觉元素位置
- ❌ 不可以写：色彩、间距、字体大小、圆角、阴影等样式规范

**原因：** 这些由UI设计师根据设计规范自行确定，不应被PRD限制

| 可以在PRD里写 | 不可以在PRD里写 |
|---------------|-----------------|
| 搜索框位于页面顶部 | 搜索框宽度560px、圆角28px |
| 卡片包含标题和摘要 | 卡片间距16px、圆角12px |
| 标签使用胶囊样式 | 标签颜色#6366F1 |
| 导航使用吸顶效果 | 导航高度64px、阴影深度 |
