## 1. 设计原则

### 1.1 核心设计原则

基于Apple HIG和Material Design提炼的四大原则：

| 原则 | 说明 | 应用场景 |
|------|------|----------|
| **清晰 (Clarity)** | 内容优先，界面元素服务于功能 | 所有界面 |
| **一致性 (Consistency)** | 相似功能使用相似设计模式 | 组件设计 |
| **层次 (Hierarchy)** | 通过视觉权重传达信息优先级 | 布局排版 |
| **克制 (Restraint)** | 只添加必要的元素 | 减法设计 |

### 1.2 设计决策判断标准

**如何判断设计是否"好看"：**

1. **功能性优先** - 设计首先服务于功能，美观是加分项
2. **一致性检验** - 与现有设计模式是否一致
3. **空白检验** - 是否有足够的呼吸空间
4. **对比检验** - 重要元素是否有足够对比度
5. **聚焦检验** - 用户视线是否被正确引导

### 1.3 风格选择指南

| 风格 | 特征 | 适用场景 |
|------|------|----------|
| **扁平化** | 无阴影、简洁色块、几何形状 | 功能型产品、数据仪表盘 |
| **新拟态** | 柔和阴影、微渐变、立体感 | 创意工具、设计软件 |
| **玻璃态** | 模糊背景、半透明、高光 | 高端产品、现代化界面 |
| **极简主义** | 大量留白、极少元素、强调 typography | 品牌官网、高端产品 |

```
错误色 Error:     #EF4444 (红色)
```

### 2.2 中性色板 (Neutral Colors)

```

200:         #F1F5F9
300:         #E2E8F0
400:         #CBD5E1
500:         #94A3B8
600:         #64748B
700:         #475569
```

### 2.3 语义色定义

| 语义 | 浅色背景 | 深色背景 | 使用场景 |
|------|----------|----------|----------|
| 成功 | #DCFCE7 | #166534 | 成功提示、完成状态 |
| 警告 | #FEF3C7 | #92400E | 警告提示、注意信息 |
| 错误 | #FEE2E2 | #991B1B | 错误提示、失败状态 |
| 信息 | #E0F2FE | #075985 | 信息提示、说明文字 |

### 2.4 暗黑模式配色

```
背景色:
  - 基础: #0F172A
  - 卡片: #1E293B
  - 悬停: #334155

文字色:
  - 主要: #F8FAFC
  - 次要: #94A3B8
  - 禁用: #475569

边框色:
  - 默认: #334155
  - 悬停: #475569
```

### 2.5 颜色使用规则

1. **主色占比** - 主色不应超过界面的20%
2. **对比度要求** - 文字与背景对比度 ≥ 4.5:1 (WCAG AA)
3. **语义一致性** - 错误不用绿色、成功不用红色

## 3. 字体规范

### 3.1 字体家族

```css
/* 中文优先 */
font-family: 
  "PingFang SC",           /* macOS */
  "Microsoft YaHei",        /* Windows */
  "Helvetica Neue",         /* 英文 */
  -apple-system,            /* iOS */
  sans-serif;               /* 回退 */

/* 等宽字体 (代码/数据) */
font-family: 
  "SF Mono",                /* macOS */
  "Cascadia Code",          /* Windows */
  "Fira Code",              /* 跨平台 */
  monospace;
```

### 3.2 字号层级

| 层级 | 名称 | 字号 | 字重 | 行高 | 使用场景 |
|------|------|------|------|------|----------|
| H1 | 大标题 | 32-48px | 700 | 1.2 | 页面主标题 |
| H2 | 标题 | 24-32px | 600 | 1.3 | 区块标题 |
| H3 | 副标题 | 18-24px | 600 | 1.4 | 卡片标题 |
| H4 | 小标题 | 16-18px | 500 | 1.4 | 列表项标题 |
| Body | 正文 | 14-16px | 400 | 1.5-1.6 | 主体内容 |
| Caption | 说明 | 12-14px | 400 | 1.4 | 辅助说明 |
| Micro | 微型 | 10-12px | 400 | 1.3 | 标签、时间戳 |

### 3.3 字重使用规范

| 字重 | 数值 | 使用场景 |
|------|------|----------|
| Thin | 100 | 装饰性文字 |
| Light | 300 | 次要辅助文字 |
| Regular | 400 | 正文内容 |
| Medium | 500 | 小标题、按钮文字 |
| Semibold | 600 | 重要标题、强调 |
| Bold | 700 | 大标题、关键词 |

## 6. 动效规范

### 6.1 时长规范

| 动画类型 | 时长 | 适用场景 |
|----------|------|----------|
| 微交互 | 100-150ms | 按钮悬停、焦点变化 |
| 组件展开 | 200-300ms | 下拉菜单、折叠面板 |
| 页面过渡 | 300-400ms | 路由切换、模态框 |
| 复杂动画 | 400-600ms | 页面加载、数据可视化 |

### 6.2 缓动曲线

```css
/* 标准缓动 (默认) */
ease: cubic-bezier(0.4, 0, 0.2, 1)

/* 进入动画 */
ease-out: cubic-bezier(0, 0, 0.2, 1)

/* 退出动画 */
ease-in: cubic-bezier(0.4, 0, 1, 1)

/* 弹性效果 */
spring: cubic-bezier(0.175, 0.885, 0.32, 1.275)

/* 匀速 (慎用) */
linear: cubic-bezier(0, 0, 1, 1)
```

### 6.3 常见动效参数

| 动效名称 | 参数 | 描述 |
|----------|------|------|
| 淡入 | opacity 0→1, 200ms | 新元素出现 |
| 淡出 | opacity 1→0, 200ms | 元素消失 |
| 上浮 | translateY(8px)→0, 200ms | 卡片悬停反馈 |
| 下压 | scale 1→0.98, 100ms | 按钮点击反馈 |
| 展开 | height auto, 300ms | 下拉展开 |
| 旋转 | rotate 0→90deg, 200ms | 图标状态切换 |

### 6.4 动效设计原则

1. **快速响应** - 交互反馈应在100ms内
2. **自然流畅** - 使用缓动曲线，避免linear
3. **克制使用** - 不是所有地方都需要动画
4. **可关闭** - 提供减少动效的选项

## 7. 图标规范

### 7.1 常用图标风格

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| **线性 (Line)** | 2px描边，圆角端点 | 现代简约界面 |
| **填充 (Filled)** | 实心，清晰明确 | 导航、按钮 |
| **双色调** | 两种颜色层次 | 强调和区分 |

### 7.2 常用尺寸

| 使用场景 | 尺寸 |
|----------|------|
| 行内图标 | 16px |
| 按钮图标 | 20px |
| 导航图标 | 24px |
| 装饰图标 | 32px |
| 独立图标 | 48px+ |

### 7.3 图标设计规范

```
描边宽度: 1.5px - 2px
圆角: 2px (端点) / 4px (转角)
最小尺寸: 16px
颜色: 跟随文字色或 #475569
悬停: 颜色变为 #2563EB
```

## 8. 布局规范

### 8.1 页面结构

```
┌──────────────────────────────────────┐
│ 顶部导航 (Header)      高度: 56-64px │
├──────────────────────────────────────┤
│                                      │
│           主内容区 (Main)             │
│         max-width: 1280px            │
│         内边距: 24px (桌面)           │
│                                      │
├──────────────────────────────────────┤
│ 底部 (Footer)        高度: auto      │
└──────────────────────────────────────┘
```

### 8.2 栅格系统

| 断点 | 宽度 | 列数 | 列宽 | 槽宽 |
|------|------|------|------|------|
| Mobile | < 640px | 4 | - | 16px |
| Tablet | 640-1024px | 8 | - | 24px |
| Desktop | 1024-1280px | 12 | - | 24px |
| Wide | > 1280px | 12 | - | 32px |

### 8.3 常用布局模式

| 布局 | 描述 | 使用场景 |
|------|------|----------|
| 单栏 | 居中内容，最大宽度 | 文档、文章 |
| 双栏 | 侧边栏 + 主内容 | 博客、设置页 |
| 三栏 | 双侧栏 + 主内容 | 仪表盘、邮件 |
| 栅格 | 多列卡片 | 产品列表、相册 |

## 9. 响应式设计

### 9.1 断点定义

```css
/* 移动端优先 */
/* Extra small: 0 - 479px */
/* Small: 480px - 767px */
/* Medium: 768px - 1023px */
/* Large: 1024px - 1279px */
/* Extra large: 1280px+ */
```

### 9.2 响应式策略

| 元素 | 移动端 | 桌面端 |
|------|--------|--------|
| 导航 | 汉堡菜单 | 横向菜单 |
| 表格 | 卡片化 | 完整表格 |
| 图片 | 宽度100% | 固定/响应 |
| 表单 | 单列 | 多列 |
| 按钮 | 全宽 | 固定宽度 |

## 10. 无障碍设计

### 10.1 基础要求

| 等级 | 对比度 | WCAG标准 |
|------|--------|----------|
| AA (最低) | 4.5:1 (正文) / 3:1 (大文字) | 1.4.3 |
| AAA (推荐) | 7:1 (正文) / 4.5:1 (大文字) | 1.4.6 |

### 10.2 交互要求

```
焦点可见: 所有可交互元素有明确焦点样式
键盘支持: Tab导航、Enter确认、Esc取消
ARIA标签: 图标按钮、复杂组件需添加aria-label
```
