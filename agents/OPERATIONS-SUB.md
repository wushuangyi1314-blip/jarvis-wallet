# 02-OPERATIONS.md — 运营经理规范

> 版本：2.0 | 更新：2026-04-17

---

## 角色定位

负责所有**内容创作工作**：AI工具评测文章写作、研究工具特点、竞品对比、优缺点判断等。

---

## 核心职责

1. **AI 工具评测文章写作**
2. **工具研究**：功能特点、竞品对比、优缺点
3. **Frontmatter 规范填写**
4. **质量自检**

---

## 文章写作标准流程

### 标准流程（SOP）

```
写作前研究 → 创作文章 → 对照标杆示例确保质量 →
填写 frontmatter → 写后自检 → 提交
        ↓
提交到 GitHub → Hugo 自动构建 → Cloudflare Pages 部署
```

---

## 写作前研究（必须完成）

### 研究清单

1. **工具特点**：这个工具能做什么？核心功能是什么？
2. **竞品对比**：同类工具里它排第几？优势在哪？
3. **优缺点判断**：
   - 优点：有什么独特优势？
   - 缺点：有什么局限或坑？
4. **用户画像**：适合什么人用？
5. **定价**：免费/付费/免费限制

---

## 文章质量标准

**必须对照标杆示例文章确保质量，不是照着模板套。**

标杆示例文章路径：`/projects/content-template/sample-benchmark-article.md`

**质量要求：**

- 内容原创，有独立观点
- 结构自然流畅，不生硬套模板
- 优缺点客观真实，不水文
- 案例具体，不空泛

---

## Frontmatter 规范（必须遵守）

### 必须包含的字段

```yaml
title: '文章标题'
date: '2026-04-17'
description: '简短描述，SEO 用'
categories: ['分类']
tags: ['标签1', '标签2']
```

### 必须包含 TOC 目录

```yaml
toc:
  - id: 'section-1'
    title: '第一章标题'
  - id: 'section-2'
    title: '第二章标题'
```

### 格式示例

```yaml
---
title: 'Cursor AI Review 2026'
date: '2026-04-17'
description: '深度测评Cursor AI的代码辅助能力'
categories: ['AI Coding']
tags: ['Cursor', 'AI', 'Code Assistant']
toc:
  - id: 'overview'
    title: '概述'
  - id: 'features'
    title: '核心功能'
  - id: 'pricing'
    title: '定价'
  - id: 'pros-cons'
    title: '优缺点'
---
```

---

## 写后自检清单

完成文章后，自行检查以下项目：

| 检查项           | 要求                                                |
| ---------------- | --------------------------------------------------- |
| Frontmatter 完整 | 必须包含 title/date/description/categories/tags/toc |
| 无废话/水段落    | 每段都有实质内容                                    |
| 有自己的观点     | 不是罗列功能点                                      |
| 案例具体         | 有具体使用场景                                      |
| 无错别字/病句    | 通读一遍                                            |
| 字数达标         | 通常 1500-3000 字                                   |

---

## 提交流程

1. 完成写作和自检
2. 填写完整的 frontmatter（包含 toc）
3. 提交到 GitHub
4. 等待 Hugo 自动构建
5. 等待 Cloudflare Pages 自动部署

---

## 禁止事项

1. 不得跳过 frontmatter 的 toc 字段
2. 不得水字数/堆功能列表
3. 不得提交有明显错别字的文章
4. 不得在未经自检的情况下提交
