## 技术验证规范（强制）

### CSS/样式验证

**必须用 curl 验证线上实际 CSS**，不是本地文件检查。

```bash
# 验证线上 CSS 是否生效
curl -s "https://aitoolreviewr.com/articles/文章slug/" | grep "CSS选择器"
```

**原因：** 本地文件正常不代表线上生效（可能 CDN/Hugo 构建/缓存问题）。

### 部署验证

| 检查项        | 方法                          |
| ------------- | ----------------------------- |
| 页面可访问    | curl 线上 URL                 |
| CSS 样式生效  | curl 线上页面 + grep CSS 类名 |
| JS 功能正常   | 控制台无报错                  |
| Hugo 构建成功 | 检查构建日志                  |

---

## Bug 严重程度定义

| 级别 | 定义                          | 处理要求     |
| ---- | ----------------------------- | ------------ |
| P0   | 页面崩溃、功能完全不可用      | 立即修复     |
| P1   | 功能异常，但有 workaround     | 当天修复     |
| P2   | 样式/交互问题，不影响核心功能 | 尽快修复     |
| P3   | 轻微问题，低频发生            | 下个版本修复 |

---

## 禁止事项

1. **不得仅检查本地文件，必须 curl 验证线上实际状态**
2. 不得将未验证的问题提交为 Bug
3. 不得省略复现步骤（无法复现的 Bug 必须标注"无法稳定复现"）
4. 不得在 Bug 未确认前提交给开发
