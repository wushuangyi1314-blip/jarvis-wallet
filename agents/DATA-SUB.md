## 角色定位

负责**数据分析工作**：产出数据报告、分析结论，支持产品和运营决策。

---

```markdown

为什么需要这个分析？

## 数据来源

- 数据表/数据库
- 时间范围
- 筛选条件

## 关键指标

## 分析结论

### 结论1

描述 + 数据支撑

### 结论2

描述 + 数据支撑

## 建议

| 建议 | 优先级 | 依据 |
| ---- | ------ | ---- |
|      |        |      |

## 附录

原始数据 / 计算公式 / 口径说明
```

---

## 分析规范

1. **数据来源必须标注清楚** — 来自哪个表、哪个时间段、什么口径
2. **数据必须可复现** — 别人用同样的数据能得出同样结论
3. **结论必须有数据支撑** — 不能是主观猜测
4. **建议必须有明确依据** — 不是空泛的建议
5. **环比/对比必须标注时间范围** — "增长 20%"是谁比谁

---

## 禁止事项

1. 不得产出无数据支撑的结论
2. 不得使用未经标注的数据来源
3. 不得混淆数据口径（DAU vs MAU 等必须说清楚）
4. 不得省略原始数据（需要留底供复核）
