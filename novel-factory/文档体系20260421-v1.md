# 小说工厂文档体系 20260421-v1

## 一、文档体系架构

小说工厂文档分为两大类：
- **通用规范文档：** 通用模板和手册，供每次小说创作时调用
- **项目文档：** 具体小说立项后产出的文档，随项目不同而不同

## 四、文档落地机制

### 核心问题：如何确保文档真正落地到Agent？

run模式的SubAgent没有记忆，每次启动都是全新的上下文。文档写得再好，Agent不知道当前该用哪个、怎么用、什么时候用。

### 解决方案：状态文件嵌入机制

每次spawn小说家Agent时，状态文件（rd-agent-state.json）必须内嵌：

```json
{
  "agent": "rd-agent",
  "task": "创作第3批第1-10章",
  "phase": "创作执行",
  "batch": "第3批",
  "required_docs": [
    "章节大纲-第3批",
    "伏笔台账",
    "人物塑造手册-第2章",
    "打斗描写手册-第3章"
  ],
  "context_summary": "主角刚突破金丹，发现宗门被灭，第2章埋下神秘玉佩伏笔",
  "active_foreshadowing": [
    "神秘玉佩-埋第2章-预计第15章回收",
    "仇人势力-埋第1章-预计第20章登场"
  ],
  "character_notes": {
    "韩立": "谨慎型-每次重大决策前犹豫3秒-当前状态：金丹初期",
    "南宫婉": "清冷型-对主角从戒备到信任-关系进度30%"
  }
}
```

### 写作手册索引说明

每个写作手册的使用说明里，必须包含：
- **调用时机：** 在什么创作场景下调用
- **调用方式：** 如何引用（例如：参考《打斗描写手册》第X章"多技能组合"小节）
- **不调用后果：** 缺少这部分会出什么质量问题
