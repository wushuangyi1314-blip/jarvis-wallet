## 一、品类 × 手册总表

### 1.1 必读手册（所有品类都需阅读）

| 手册 | 用途 | 调用时机 |
|------|------|---------|
| 13a-人物塑造手册v2 | 人物性格/心理/成长/对白 | 创作人物戏份前，必读 |
| 13h-爽点设计手册v2 | 升级快感/打脸爽点/情感共鸣点 | 铺垫爽点/高潮前，必读 |

### 1.2 品类对应打斗手册

| 品类 | 套路口径 | 战斗手册 | 说明 |
|------|---------|---------|------|
| **玄幻仙侠** | 02a | 13b修仙玄幻 + 13a打斗基础 | 修仙战斗必读13b |
| **都市言情** | 02b | 13e都市现代战斗 | 感情线为主，战斗场景少，按需调用 |
| **穿越历史** | 02c | 13b修仙玄幻（古代背景）或 13e都市（现代背景） | 按实际时代背景选择 |
| **虚拟网游** | 02d | 13d网游打斗 + 13a打斗基础 | 网游战斗必读13d |
| **末日废土** | 02e | 13f末日生存战斗 + 13a打斗基础 | 末日战斗必读13f |
| **科幻机甲** | 02f | 13c科幻机甲 + 13a打斗基础 | 机甲战斗必读13c |
| **悬疑灵异** | 02g | 极少，视情况 | 侧重氛围，无专门战斗手册 |
| **现代都市** | 02h | 13e都市现代战斗 | 都市异能/商战/鉴宝等 |

## 三、战斗场景调用顺序（标准流程）

```
每次战斗场景创作前，按以下顺序调用：

【第一步】13a 打斗基础理论
         ↓ 必读，建立战斗框架
【第二步】品类对应战斗手册
         玄幻→13b | 网游→13d | 科幻→13c | 都市→13e | 末日→13f
         ↓
【第三步】13g 战斗写作手册
         （战斗通用技巧+心理/情绪/恐惧描写）
         ↓
【第四步】若涉及修仙/玄幻元素
         加读13b（六大流派：肉身/法术/阵法/法宝/精神/功法）
```

## 五、状态文件 required_docs 填写规范

每次 spawn SubAgent 时，状态文件必须填写 `required_docs`，示例：

### 玄幻仙侠项目战斗场景
```json
{
  "novel_category": "玄幻仙侠",
  "required_docs": {
    "must_read": [
      "13a-人物塑造手册v2",
      "13h-爽点设计手册v2"
    ],
    "combat": [
      "13a-打斗基础理论",
      "13b-修仙玄幻打斗",
      "13g-战斗写作手册"
    ],
    "character": [
      "13a-人物塑造手册v2"
    ],
    "antagonist": [
      "13c-反派设计手册v2"
    ]
  }
}
```

## 六、调用检查清单

每次创作前，确认以下问题：

- [ ] 主品类已确认 → 对应套路口径（02x）已确定
- [ ] 必读手册（13a人物 + 13h爽点）已加入 required_docs
- [ ] 战斗场景需要哪个手册？→ 已查品类对应表确认
- [ ] 是否需要 13a 打斗基础？→ 大多数战斗场景需要
- [ ] 是否需要 13g 战斗写作？→ 心理/情绪描写时需要
- [ ] 反派首次出场？→ 需加入 13c反派设计手册
