## 整体结构

将11步重组为**5个阶段**，每个阶段有明确的输入、输出、决策点：
- 阶段一：创意立项（步骤1-3）
- 阶段二：规划设计（步骤4-6）
- 阶段三：纲目制定（步骤7-9）
- 阶段四：创作执行（步骤10）← 本次更新
- 阶段五：定稿审读（步骤11）

每个阶段内可小循环，跨阶段不可跳跃，必须按序推进。

---

## 阶段一：创意立项

### 步骤1｜创意 → 主线故事

**输入：**
- 品类（玄幻/都市/网游等）
- 套路（凡人流/退婚流/系统流等）
- 文笔风格（小白文/老白文/精品文等）
- 目标字数（300万字）
- 用户原始创意描述

**输出：《主线故事》文档，包含：**
- 一句话核心卖点（≤30字）
- 三句话故事简介（≤100字）
- 主线剧情（≤500字，描述主角从何处来、遇何事、有何目标、核心障碍是谁）
- 核心冲突（主角 vs 谁/什么，最终解决什么问题）
- 故事气质定位（热血/压抑/轻松/黑暗）

**阿呆动作：** 丰满用户原始创意，补充上述要素，形成完整主线故事。输出后主动告知用户。

---

### 步骤2｜主线故事 → 立项评估

**输入：** 《主线故事》

**输出：** 评估结论

| 结论 | 含义 | 后续动作 |
|------|------|---------|
| ✅ 立项 | 故事符合预期 | 进入步骤3 |
| 🔄 小改 | 局部调整，方向正确 | 回到步骤1局部修改，输出《主线故事》v2 |
| ❌ 重写 | 方向性偏差 | 回到步骤1重新输出《主线故事》v1 |

**注意：** 小改/重写必须有版本标注（v1/v2）。

---

### 步骤3｜主线故事 → 选题验证

**输入：** 《主线故事》

**阿呆执行验证，输出《选题验证报告》，包含以下维度：**

| 验证维度 | 验证内容 | 通过标准 |
|---------|---------|---------|
| 字数可达性 | 当前主线能否撑到目标字数 | 有≥3个可扩展的高光节点/副本 |
| 差异化 | 与同类标杆作品的区分度 | 核心卖点与竞品有明显差异 |
| 市场受欢迎程度 | 同类作品历史表现 | 近3年同套路作品平均追读率 |
| 主角代入感 | 主角目标/困境是否让读者有共鸣 | 主角困境≥3个，情绪共鸣点≥2个 |
| 商业化潜力 | IP衍生/改编可能性 | 至少1个可衍生元素 |
| 节奏可行性 | 每章能否撑住2000-4000字 | 有≥5个可拆分章节的核心情节点 |

**验证结论：**

| 结论 | 含义 | 后续动作 |
|------|------|---------|
| ✅ 通过 | 选题成立 | 进入阶段二 |
| ⚠️ 有风险 | 有短板但可弥补 | 阿呆提出弥补方案，用户确认后进入阶段二 |
| ❌ 不通过 | 根本性问题 | 回到步骤1重新输出《主线故事》 |

---

## 阶段二：规划设计

### 步骤4｜主线故事 → 核心设定

**输入：** 《主线故事》+ 品类 + 套路 + 文笔风格 + 目标字数

**输出：《核心设定》文档，包含：**

| 模块 | 内容要求 |
|------|---------|
| 世界规则 | 几重世界/位面，主世界特征，力量上限 |
| 力量体系 | 存在哪几种力量，核心力量是什么，高层战力特征 |
| 核心矛盾 | 主角要解决什么问题，最大障碍是什么，对手势力是谁 |
| 主角设定 | 基础信息（姓名/性别/年龄/出身）+ 性格特质 + 人物弧光 + 核心执念 + 能力设定（必须包含一项独特金手指） |
| 女主设定 | 单女主/多女主/无女主，女主类型，基础信息，与主角关系定位 |
| 其他重要角色 | 1-2个核心配角简要设定（后续随故事补充） |

**阿呆动作：** 根据品类和套路，自动调取对应套路模板，套入主线故事输出核心设定。

---

### 步骤5｜核心设定 → 初版卷纲

**输入：** 《主线故事》+ 《核心设定》

**输出：《初版卷纲》，要求：**
- 按卷纲模板输出（每卷独立章节概要）
- 每卷有卷名、卷核心事件、卷结尾悬念
- 规划总卷数，估算每卷字数区间（建议每卷25-40万字）
- 卷与卷之间有明确的情节递进和悬念连接

---

### 步骤6｜核心设定 → 小说设定

**输入：** 《主线故事》+ 《核心设定》+ 品类 + 套路

**输出：《小说设定》文档**

**重要规则：设定文档覆盖更新，不留旧版本**
- 文件名不带版本后缀（如 `机甲详细设定.md`，不写 `机甲详细设定v8.md`）
- 每次覆盖前，先 git commit 原版（留可回退）
- 版本历史记录在文档内的版本历史表里
- 同一设定文档只保留1个文件，不留 v1/v2/v8 共存的情况
- **定期清理**：每次创作流启动前（步骤4前），检查设定文档是否有多余旧版本，有则删除或合并

根据品类不同，设定分为**通用部分**（所有品类都要填）和**分类特殊部分**（按品类选择填写）：

**通用部分（必填）：**
- 世界规则设定
- 世界矛盾设定
- 力量体系设定
- 社会体系设定
- 等级体系
- 势力设定
- 种族设定
- 地图设定
- 人物设定（主角+主要反派+主要配角）
- 关系网设定
- 主线/支线故事任务设定
- 升级路线/奇遇设定

**分类特殊部分（按品类选填）：**

| 品类 | 特殊设定项 |
|------|-----------|
| 网游 | 技能设定、装备设定、道具设定、公会/团队设定、副本设定 |
| 玄幻/仙侠 | 功法设定、灵根/资质设定、炼丹/炼器设定、宗门设定 |
| 都市 | 职业设定、商业设定、身份设定（隐藏身份/双重身份） |
| 末日/废土 | 丧尸/怪物设定、基地设定、资源设定、阵营设定 |
| 科幻 | 科技设定、星际设定、舰船/武器设定 |
| 历史 | 时代背景设定、势力/门派设定、武学设定 |

---

### 步骤6.5｜小说设定 → 设定索引（自动创建）

**输入：** 《小说设定》文档列表

**输出：** 自动创建以下两个索引文档到项目目录下：

**A. 《设定索引.md》**
- 核心术语按字母A-Z分类索引
- 每条索引包含：术语名、分类、定义/参数、文档位置（文件名#行号）
- 创作场景快速定位（战斗/商业/航行/政治/人物）

**B. 《速查手册.md》**
- 压缩到3-5页的核心数据
- 包含：关键参数一览表、价格速查表、势力关系图、人物关系速查

**自动触发条件：** 每次完成《小说设定》文档后自动执行，无需人工触发

**索引维护规则：** 当新增/修改设定文档后，同步更新索引文档的对应条目

---

## 阶段三：纲目制定

### 步骤7｜小说设定 → 终版卷纲

**输入：** 《小说设定》+ 《初版卷纲》

**输出：《终版卷纲》**

- 基于初版卷纲，根据小说设定补充的信息进行修订
- 每卷增加更详细的情节走向说明
- 补充卷内核心情节点的数量（建议每卷≥8个核心情节点）
- 确认伏笔埋设位置（前卷埋、后卷收）
- 版本标注：v1（初版）→ v2（终版）

---

### 步骤8｜终版卷纲 → 小说大纲

**输入：** 《终版卷纲》+ 《小说设定》

**输出：《小说大纲》**

按小说大纲模板输出，包含：
- 全书主线剧情（按时间线展开，5000-10000字）
- 每卷核心事件列表
- 主角成长弧光（起点→转折→高潮→结局）
- 重要配角的独立故事线
- 伏笔分布总图（标注埋入卷/回收卷）
- 爽点/高潮点排布（全书≥10个高光节点）

---

### 步骤9｜小说大纲 → 章节大纲 + 质量保障文档

**输入：** 《小说大纲》

**输出：**
1. **《章节大纲》** — 按章节大纲模板，每章包含：
   - 章号+章名
   - 本章核心事件（≤50字）
   - 本章出场人物
   - 本章信息量指标（推进/埋伏笔/回收/高潮）
   - 与上下章的衔接点

2. **《质量保障文档》** — 包含：
   - **伏笔台账（预填）**：从章节大纲中提取所有伏笔提示，生成伏笔条目，初始状态全部为"未埋入"
   - 人物设定一致性核查表
   - 关系网维护表
   - 上下文连贯性检查清单
   - 目标字数跟踪表（每卷/每批实际vs目标）
   - **本批设定引入任务清单（预填）**：根据章节大纲，列出每批需要引入的设定项

3. **《写作上下文》** — 第1批写作前阿呆初始化，后续每批写作后更新，包含：
   - 上批结尾情节（首批写作上下文无"上批"，见下方初始化规则）
   - 人物当前状态
   - 本批起点

#### 步骤9.5｜质量保障文档伏笔台账预填规则（新增）

**伏笔台账预填时机：** 步骤9产出质量保障文档时，伏笔台账必须预填完毕，不能留空。

**预填操作：**
1. 遍历章节大纲，找到所有标注"埋伏笔"的章节
2. 为每个伏笔生成一条台账条目：
   - 序号：自动编号（FS-J-001/FS-Z-001/FS-Y-001）
   - 伏笔名称：从伏笔提示提取或根据上下文拟定
   - 埋入章节：章节大纲中标注的章节号
   - 计划回收章节：章节大纲中标注的回收章节号（如无标注，则根据伏笔重要性估算）
   - **状态：初始为"未埋入"**
3. 无伏笔提示的章节，在台账中注明"本章无伏笔任务"

**伏笔台账初始状态示例：**

```
| 序号 | 伏笔名称 | 埋入章节 | 计划回收 | 状态 |
| FS-J-001 | 基因匹配度97.3% | 第10章 | 第250章 | 未埋入 |
| FS-Z-001 | 神秘机甲的来源 | 第15章 | 第80章 | 未埋入 |
```

#### 步骤9.6｜本批设定引入任务清单预填规则（新增）

**时机：** 步骤9产出质量保障文档时，设定引入任务清单必须预填完毕。

**预填操作：**
1. 遍历章节大纲，识别每批涉及的设定域（如机甲战斗/虫族出现/星际航行等）
2. 根据设定域，标注每批需要引入的设定项
3. 格式：

```
### 第1批设定引入任务
| 设定项 | 引入章节 | 来源文档 | 注意事项 |
| 机甲能量核心原理 | 第5章 | 机甲详细设定#第3章 | 只需引入概念，不要展开太多 |

### 第2批设定引入任务
| 设定项 | 引入章节 | 来源文档 | 注意事项 |
| 虫族女王特性 | 第15章 | 虫族详细设定 | 对应第1批埋入的虫族伏笔 |
```

#### 写作上下文初始化规则（第1批专用的Bootstrap格式）

**问题：** 第1批写作上下文没有"上批"可以承接，如何初始化？

**解决方案：** 第1批写作上下文的"上批结尾摘要"区块，用"项目启动"作为虚拟上批：

```
## 上批结尾摘要（项目启动 → 第1批）

### 情节承接

项目启动状态：
> 故事从第1章开始，主角[姓名]处于[初始状态/处境]，故事核心矛盾是[核心冲突]。

本批开头（第1章）需要交代：
> 主角的背景、当前处境、核心目标

### 人物状态（初始状态）

主角当前状态：
- 境界/实力：[初始境界]
- 位置：[初始位置]
- 目标：[初始目标]
- 关键物品：[初始携带物品]

女主当前状态：
- 境界/实力：[初始境界]
- 位置：[初始位置/尚未出场]
- 与主角关系进度：尚未建立

### 势力/世界状态（初始状态）

世界局势：[初始世界主要矛盾]
主要势力动态：[各势力初始状态]
```

**更新规则：** 每批写完后，在写作上下文中追加新的"上批结尾摘要"，而非覆盖历史记录。

## 阶段四：创作执行

### 步骤10｜章节大纲 → 小说初版

**执行者：** 阿呆直接写作（不再使用SubAgent）

**输入：** 《章节大纲》+ 《质量保障文档》+ 《写作上下文》+ 《设定索引》+ 《小说写作手册》

**输出：** 分批《小说初版》章节

---

#### 三层文档结构（重要）

步骤10写作过程中，涉及以下三层文档：

| 文件 | 内容 | 阿呆读取频率 |
|------|------|------------|
| `XX-质量保障文档.md`（主文档） | 最近3批汇总（伏笔/人物状态/任务），固定大小不膨胀 | **每次写新章必读** |
| `XX-写作上下文.md`（主文档） | 最近3批汇总（情节承接/人物状态），固定大小不膨胀 | **每次写新章必读** |
| `批次记录/批次N-质量记录.md` | 本批10章各自的变更（YAML追加），完整历史存档 | 有问题时查 |

**批次记录文件命名：**
```
批次记录/
  01-批次1-质量记录.md   ← 第1批
  02-批次2-质量记录.md   ← 第2批
  ...
  77-批次77-质量记录.md  ← 第77批（775章/10章一批）
```

**主文档滚动规则：**
- 只保留最近3批的汇总信息
- 第4批写完时，第1批的详情已移入批次记录，主文档移除第1批汇总、加入第4批汇总
- 始终保持：最近3批汇总

---

#### 核心原则

1. **写完 ≠ 结束，写完 + 每章更新文档 + 去AI味 + 用户确认才算完成**
2. **大纲是参考，不是命令；写出更好的内容，更新大纲**
3. **阿呆评估「更好」，用户拍板「确认」**
4. **每批10章为最小写作单元，每章单独流程**
5. **伏笔数据以质量保障文档为唯一数据源**
6. **设定引入有规则，查设定有时机，不能凭空编造**

---

#### 三个核心文档（每批必读）

| 文档 | 作用 | 读写规则 |
|------|------|---------|
| `XX-章节大纲.md` | 写作提纲：章号 + 核心事件 + 伏笔提示 | **只读不写** |
| `XX-质量保障文档.md`（主文档） | 最近3批伏笔汇总 + 人物状态 + 本批任务 | **每章追加写入** |
| `XX-写作上下文.md`（主文档） | 最近3批情节承接 + 人物状态 | **每章追加写入** |

---

#### 每章写作流程（6步）

每章写完，立即执行以下6步，再进入下一章：

```
[1/6] 冲突检测（写前）
  → 本章新内容与已有设定/人物状态对照
  → 有冲突则记录到批次记录问题区块

[2/6] 读章节大纲（本章）
  → 确认本章核心事件
  → 确认伏笔提示
  → 识别本章涉及的设定域

[3/6] 读质量保障文档 + 写作上下文（主文档）
  → 确认当前活跃伏笔状态
  → 确认人物当前状态
  → 确认本章设定引入任务

[4/6] 写本章正文
  → 专注写正文
  → 如需查设定：用 [查：XXX] 标记，继续写

[5/6] ★ Checklist-α 自检（本章写完后）★
  → 必须通过全部8项检查，方可进入下一步
  → 有未通过项：记录到批次记录，重写对应段落，再自检

[6/6] 输出元数据YAML（本章写完后）
  → 产出YAML结构（见下方格式）
  → 追加写入「批次记录/批次N-质量记录.md」
  → 同时更新主文档（质量保障文档 + 写作上下文）的最近3批汇总区块
```

#### Checklist-α（每章自检清单）

每章写完，必须逐项检查：

| # | 检查项 | 合格标准 |
|---|--------|---------|
| 1 | 时间线 | 本章事件与前章时间间隔合理，无逻辑跳跃 |
| 2 | 空间一致性 | 场景转换交代清楚，人物移动符合物理逻辑 |
| 3 | 人物一致性 | 对话风格与前章一致，行为符合当前人设 |
| 4 | 伏笔处理 | 标记为"本章需处理"的伏笔全部提及 |
| 5 | 情绪连贯 | 情绪节拍自然呈现，无突兀转变 |
| 6 | 物品一致性 | inventory中物品出现合理，新增物品已记录 |
| 7 | 对话人物语音 | 每个角色说话方式有辨识度，不通用化 |
| 8 | 五感描写 | 每个场景至少2种感官描写（视/听/嗅/味/触） |

**未通过处理：** 记录到批次记录对应章节条目下，重写后重新自检。

#### 每批收尾流程（10章全部写完后）

```
[1/4] 设定查漏
  → 检查本批所有 [查：XXX] 标记
  → 确认都已查阅，无遗漏

[2/4] 去AI味
  → 应用 humanize-chinese-2-0-0 技能
  → 对本批10章正文逐一去AI味
  → 完成后注明"已去AI味"

[3/4] 批次记录归档
  → 将本批10章的YAML记录从内存写入「批次记录/批次N-质量记录.md」
  → 更新主文档：移除第1批汇总，加入本批汇总（保持最近3批）

[4/4] 向用户提交批次摘要
  → 阿呆向用户报告本批写作结果，格式：
    第N批（第X-Y章）写完
    - 符合大纲：是/否
    - 有偏离：第X章，内容是...，原因是...
    - 偏离是否更好：待用户判断
    - 已去AI味：是
  → 用户确认后，再开始下一批
  → 如有偏离待用户判断：停止下一批，等待用户反馈
```

---

#### 用户确认机制

**确认时限：** 批次摘要提交后，用户超过24小时无回复，阿呆停止下一批写作。

**超时不回复的处理：**
1. 阿呆在批次摘要中注明"请在24小时内确认，如超时未回复我将暂停下一批"
2. 超时后：停止下一批，告知用户"已超时，等待您的确认后再继续"
3. 用户恢复确认后：阿呆继续下一批，无需重新启动

**例外：** 如用户提前说明"不用等确认，直接继续"，则跳过此机制。

#### 冲突检测规则

每章写作前，检测本章新内容是否与已有设定冲突：

| 冲突类型 | 检测方法 | 发现后处理 |
|---------|---------|---------|
| 人物设定矛盾 | 本章人物行为是否符合已确立的性格/能力 | 记录到issues区块 |
| 场景/地点矛盾 | 本章地点描述是否与前文一致 | 记录到issues区块 |
| 物品使用矛盾 | 本章物品是否在inventory中 | 记录到issues区块 |
| 时间线矛盾 | 本章时间推进是否合理 | 记录到issues区块 |
| 力量体系矛盾 | 本章战斗/能力是否与设定等级一致 | 记录到issues区块 |

#### 失败记录机制

发现问题立即标记，最终汇总到批次记录：

```
发现问题时 → 在本章YAML的issues区块标记
  - type: 问题类型（穿帮/矛盾/节奏失控/情节重复/其他）
  - 严重程度: low/medium/high/critical
  - 描述: 具体问题
  - 处理: 如何修复

批级汇总 → 本批10章全部写完后，在批次记录文件末尾追加：
  ## 批次N问题汇总
  - 所有issues按严重程度排序
  - critical/high需要阿呆提出解决方案
  - medium/low可选择性处理
```

#### 设定域识别规则

每批写作前，根据章节大纲识别涉及哪些设定域：

| 场景类型 | 触发设定域 | 查询位置 |
|---------|----------|---------|
| 机甲战斗 | 机甲能力参数/武器系统 | 设定索引→机甲详细设定 |
| 虫族出现 | 虫族特性/分类/弱点 | 设定索引→虫族详细设定 |
| 星际航行 | 星域地图/航行能力 | 设定索引→势力地图 |
| 政治/谈判 | 势力关系/历史渊源 | 设定索引→势力地图 |
| 经济交易 | 货币体系/价格换算 | 设定索引→经济货币体系 |
| AI/量子相关 | 科技原理/AI特性 | 设定索引→科技树设定 |
| 历史回顾 | 帝国消亡/战争真相 | 设定索引→世界规则设定 |
| 人物出场 | 角色背景/关系网 | 设定索引→人物关系设定 |

**查询规则：**

| 情况 | 处理方式 |
|------|---------|
| 已熟悉 | 边写边用 |
| 记忆模糊 | 先查再写 |
| 完全不知道 | 先查再写 |
| 写作中遇到不确定 | 用 `[查：XXX]` 标记，本章结束后统一查 |

**设定引入质量标准：**
- **适时**：情节需要时引入，不提前不落后
- **准确**：设定内容与文档一致，不能凭空编造
- **恰当**：通过动作/对话/反应引入，不通过旁白说明

#### 创意偏离处理

当阿呆觉得「写出了比大纲更好的内容」时：

1. 阿呆提出建议：
   「第X章大纲 vs 实际写法对比
   - 大纲写法：...
   - 实际写法：...
   - 原因：逻辑更顺/情感更强/冲突更烈
   - 建议：更新章节大纲为...」

2. 用户决定：
   ✅ 确认更新 → 阿呆更新章节大纲
   ❌ 保留大纲 → 保持原大纲
   🔄 折中方案 → 用户说明，阿呆执行

3. 只有用户确认后，才能更新章节大纲

## 阶段五：定稿审读

### 步骤11｜小说初版 → 小说定稿

**输入：** 《小说初版》（分批）+ 《质量保障文档》

**阿呆动作：** 整批审读后给出修改建议，用户确认后输出定稿

**审读检查清单：**

| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 伏笔回收 | 100%回收，有记录的时间差≤30章 | 打回修改 |
| 角色行为逻辑 | 行为符合人设，无突兀转变 | 打回修改单章 |
| 名称一致性 | 全文无名称错误 | 统一修正 |
| 信息一致性 | 与设定文档无冲突 | 统一修正 |
| 情节连贯性 | 章与章之间衔接自然 | 打回修改 |
| 错别字/语病 | 每章≤3处 | 批量修正 |
| 信息密度 | 无水章（每章有实质推进） | 打回修改 |
| 章节钩子 | 末尾钩子强度≥3分（5分制） | 加强或补充 |
| 节奏感 | 高潮点间隔≤15章 | 重新排布 |

**输出：** 《小说定稿》（标注版本v定稿）
