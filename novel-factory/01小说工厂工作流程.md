## 整体结构

将11步重组为**5个阶段**，每个阶段有明确的输入、输出、决策点：
- 阶段一：创意立项（步骤1-3）
- 阶段二：规划设计（步骤4-6）
- 阶段三：纲目制定（步骤7-9）
- 阶段四：创作执行（步骤10）
- 阶段五：定稿审读（步骤11）

每个阶段内可小循环，跨阶段不可跳跃，必须按序推进。

---

## 阶段一：创意立项

### 步骤1｜创意 → 主线故事

**输入：**
- 品类（玄幻/都市/网游等）
- 套路（凡人流/退婚流/系统流等）
- 文笔风格（小白文/老白文/精品文等）
- 目标字数（300万字）
- 用户原始创意描述

**输出：《主线故事》文档，包含：**
- 一句话核心卖点（≤30字）
- 三句话故事简介（≤100字）
- 主线剧情（≤500字，描述主角从何处来、遇何事、有何目标、核心障碍是谁）
- 核心冲突（主角 vs 谁/什么，最终解决什么问题）
- 故事气质定位（热血/压抑/轻松/黑暗）

**阿呆动作：** 丰满用户原始创意，补充上述要素，形成完整主线故事。输出后主动告知用户。

---

### 步骤2｜主线故事 → 立项评估

**输入：** 《主线故事》

**输出：** 评估结论

| 结论 | 含义 | 后续动作 |
|------|------|---------|
| ✅ 立项 | 故事符合预期 | 进入步骤3 |
| 🔄 小改 | 局部调整，方向正确 | 回到步骤1局部修改，输出《主线故事》v2 |
| ❌ 重写 | 方向性偏差 | 回到步骤1重新输出《主线故事》v1 |

**注意：** 小改/重写必须有版本标注（v1/v2），阿呆需在状态文件中记录当前版本号。

---

### 步骤3｜主线故事 → 选题验证

**输入：** 《主线故事》

**阿呆执行验证，输出《选题验证报告》，包含以下维度：**

| 验证维度 | 验证内容 | 通过标准 |
|---------|---------|---------|
| 字数可达性 | 当前主线能否撑到目标字数 | 有≥3个可扩展的高光节点/副本 |
| 差异化 | 与同类标杆作品的区分度 | 核心卖点与竞品有明显差异 |
| 市场受欢迎程度 | 同类作品历史表现 | 近3年同套路作品平均追读率 |
| 主角代入感 | 主角目标/困境是否让读者有共鸣 | 主角困境≥3个，情绪共鸣点≥2个 |
| 商业化潜力 | IP衍生/改编可能性 | 至少1个可衍生元素 |
| 节奏可行性 | 每章能否撑住2000-4000字 | 有≥5个可拆分章节的核心情节点 |

**补充验证（阿呆负责，可调用数据查询）：**
- 目标读者画像匹配度（年龄/性别/偏好）
- 竞品分析（同类作品数量、头部作品数据）
- 金手指/系统新颖程度（避免与近期爆款高度雷同）

**验证结论：**

| 结论 | 含义 | 后续动作 |
|------|------|---------|
| ✅ 通过 | 选题成立 | 进入阶段二 |
| ⚠️ 有风险 | 有短板但可弥补 | 阿呆提出弥补方案，用户确认后进入阶段二 |
| ❌ 不通过 | 根本性问题 | 回到步骤1重新输出《主线故事》 |

---

## 阶段二：规划设计

### 步骤4｜主线故事 → 核心设定

**输入：** 《主线故事》+ 品类 + 套路 + 文笔风格 + 目标字数

**输出：《核心设定》文档，包含：**

| 模块 | 内容要求 |
|------|---------|
| 世界规则 | 几重世界/位面，主世界特征，力量上限 |
| 力量体系 | 存在哪几种力量，核心力量是什么，高层战力特征 |
| 核心矛盾 | 主角要解决什么问题，最大障碍是什么，对手势力是谁 |
| 主角设定 | 基础信息（姓名/性别/年龄/出身）+ 性格特质 + 人物弧光 + 核心执念 + 能力设定（必须包含一项独特金手指） |
| 女主设定 | 单女主/多女主/无女主，女主类型，基础信息，与主角关系定位 |
| 其他重要角色 | 1-2个核心配角简要设定（后续随故事补充） |

**阿呆动作：** 根据品类和套路，自动调取对应套路模板，套入主线故事输出核心设定。

---

### 步骤5｜核心设定 → 初版卷纲

**输入：** 《主线故事》+ 《核心设定》

**输出：《初版卷纲》，要求：**
- 按卷纲模板输出（每卷独立章节概要）
- 每卷有卷名、卷核心事件、卷结尾悬念
- 规划总卷数，估算每卷字数区间（建议每卷25-40万字）
- 卷与卷之间有明确的情节递进和悬念连接

---

### 步骤6｜初版卷纲 → 小说设定

**输入：** 《初版卷纲》+ 《核心设定》+ 品类 + 套路

**输出：《小说设定》文档**

根据品类不同，设定分为**通用部分**（所有品类都要填）和**分类特殊部分**（按品类选择填写）：

**通用部分（必填）：**
- 世界规则设定
- 世界矛盾设定
- 力量体系设定
- 社会体系设定
- 等级体系
- 势力设定
- 种族设定
- 地图设定
- 人物设定（主角+主要反派+主要配角）
- 关系网设定
- 主线/支线故事任务设定
- 升级路线/奇遇设定

**分类特殊部分（按品类选填）：**

| 品类 | 特殊设定项 |
|------|-----------|
| 网游 | 技能设定、装备设定、道具设定、公会/团队设定、副本设定 |
| 玄幻/仙侠 | 功法设定、灵根/资质设定、炼丹/炼器设定、宗门设定 |
| 都市 | 职业设定、商业设定、身份设定（隐藏身份/双重身份） |
| 末日/废土 | 丧尸/怪物设定、基地设定、资源设定、阵营设定 |
| 科幻 | 科技设定、星际设定、舰船/武器设定 |
| 历史 | 时代背景设定、势力/门派设定、武学设定 |

---

### 步骤6.5｜小说设定 → 设定索引（自动创建）

**输入：** 《小说设定》文档列表

**输出：** 自动创建以下两个索引文档到 `项目文档模板/` 目录：

**A. 《设定索引.md》**
- 核心术语按字母A-Z分类索引
- 每条索引包含：术语名、分类、定义/参数、文档位置（文件名#行号）
- 涵盖：机甲、武器、材料、战舰、货币、势力、种族、星域、人物、任务
- 创作场景快速定位（战斗/商业/航行/政治/人物）

**B. 《速查手册.md》**
- 压缩到3-5页的核心数据
- 包含：机甲参数一览表、战舰参数一览表、价格速查表、势力关系图、人物关系速查、星域速查、科技等级速查
- 创作时一目了然，不用跳转多个文档

**自动触发条件：** 每次完成《小说设定》文档后自动执行，无需人工触发

**索引维护规则：** 当新增/修改设定文档后，同步更新索引文档的对应条目

---

## 阶段三：纲目制定

### 步骤7｜小说设定 → 终版卷纲

**输入：** 《小说设定》+ 《初版卷纲》

**输出：《终版卷纲》**

- 基于初版卷纲，根据小说设定补充的信息进行修订
- 每卷增加更详细的情节走向说明
- 补充卷内核心情节点的数量（建议每卷≥8个核心情节点）
- 确认伏笔埋设位置（前卷埋、后卷收）
- 版本标注：v1（初版）→ v2（终版）

---

### 步骤8｜终版卷纲 → 小说大纲

**输入：** 《终版卷纲》+ 《小说设定》

**输出：《小说大纲》**

按小说大纲模板输出，包含：
- 全书主线剧情（按时间线展开，5000-10000字）
- 每卷核心事件列表
- 主角成长弧光（起点→转折→高潮→结局）
- 重要配角的独立故事线
- 伏笔分布总图（标注埋入卷/回收卷）
- 爽点/高潮点排布（全书≥10个高光节点）

---

### 步骤9｜小说大纲 → 章节大纲 + 质量保障文档

**输入：** 《小说大纲》

**输出：**
1. **《章节大纲》** — 按章节大纲模板，每章包含：
   - 章号+章名
   - 本章核心事件（≤50字）
   - 本章出场人物
   - 本章信息量指标（推进/埋伏笔/回收/高潮）
   - 与上下章的衔接点

2. **《质量保障文档》** — 包含：
   - 伏笔台账（所有伏笔列表：名称+埋入章节+回收章节+状态）
   - 人物设定一致性核查表（主角/女主/主要配角的核心特质清单）
   - 关系网维护表（角色间关系变化记录）
   - 上下文连贯性检查清单
   - 目标字数跟踪表（每卷/每批实际vs目标）

---

## 阶段四：创作执行

### 步骤10｜章节大纲 + 伏笔台账 → 小说初版

**输入：** 《章节大纲》+ 《伏笔台账》+ 《质量保障文档》+ 《小说写作手册》

**输出：** 分批《小说初版》章节

**创作要求：**

1. **分批输出：** 每批10章，循环创作
2. **每批前置动作：** 阿呆读取状态文件，确认当前批次+伏笔状态+人物设定
3. **每批前置输出：** 本批前情概要（30字以内），防止上下文断裂
4. **创作过程中：**
   - 人物塑造：参考《人物塑造手册》
   - 打斗/场面描写：参考《打斗描写手册》
   - 反派设计：参考《反派设计手册》
   - 爽点设计：参考《爽点设计手册》
5. **伏笔管理：** 埋入伏笔时实时更新伏笔台账（状态文件）
6. **质量保障：** 每批输出后，对照质量保障文档做自检
7. **中文去AI味：** 应用humanize-chinese-2-0-0技能
8. **金手指使用原则：** 先抑后扬，主角开篇不能过强，每场胜利前必须有充分铺垫

**状态文件必须包含的字段：**
```json
{
  "agent": "rd-agent",
  "task": "当前任务描述",
  "phase": "当前阶段",
  "batch": "第N批",
  "required_docs": ["章节大纲", "伏笔台账", "人物塑造手册"],
  "context_summary": "前N批核心情节点（50字以内）",
  "active_foreshadowing": ["伏笔A-埋第X章-预计第Y章回收"],
  "character_notes": {"主角名": "性格要点", "女主名": "关系定位"}
}
```

## 阶段五：定稿审读

### 步骤11｜小说初版 → 小说定稿

**输入：** 《小说初版》（分批）+ 《伏笔台账》+ 《质量保障文档》

**阿呆动作：** 整批审读后给出修改建议，用户确认后输出定稿

**审读检查清单：**

| 检查项 | 合格标准 | 不合格处理 |
|--------|---------|-----------|
| 伏笔回收 | 100%回收，有记录的时间差≤30章 | 打回修改 |
| 角色行为逻辑 | 行为符合人设，无突兀转变 | 打回修改单章 |
| 名称一致性 | 全文无名称错误 | 统一修正 |
| 信息一致性 | 与设定文档无冲突 | 统一修正 |
| 情节连贯性 | 章与章之间衔接自然 | 打回修改 |
| 错别字/语病 | 每章≤3处 | 批量修正 |
| 信息密度 | 无水章（每章有实质推进） | 打回修改 |
| 章节钩子 | 末尾钩子强度≥3分（5分制） | 加强或补充 |
| 节奏感 | 高潮点间隔≤15章 | 重新排布 |

**输出：** 《小说定稿》（标注版本v定稿）

## 流程版本控制规则

| 规则 | 说明 |
|------|------|
| 每次重大修改 | 版本号+1（v1→v2） |
| 小改 | 在版本后加补丁（v1.1） |
| 版本记录 | 状态文件中记录版本历史和变更原因 |
| 回退 | 任何阶段发现重大问题，可回退到上一版本重新来过 |
