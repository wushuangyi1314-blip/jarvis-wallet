## 文件命名规范

### 章节文件
```
{序号}-{章号}-第{章号}章-{章名}.md

示例：
001-第1章-绞肉机.md
002-第2章-炮灰的命运.md
025-第25章-序章终.md
026-第26章-蛮荒之地.md
150-第150章-第一卷尾.md
```

### 版本记录文件
```
v{版本}-{批次}-{卷号}-{范围}-{日期}.md

示例：
v1.0-第1批-序章-第1-10章-20260423.md
v1.0-第1批-序章-第11-25章-20260425.md
```

## 创作流程

1. **创作**：SubAgent按批次创作章节（每批10章）
2. **存放**：完成后存入 `01-章节草稿/{卷号}/`
3. **审核**：阿呆审核后标记为定稿
4. **移动**：定稿移入 `02-定稿/{卷号}/`
5. **记录**：版本记录存入 `03-版本记录/`

## 展示规则

### 每批完成（10章）
- 使用 lightclaw 上传打包文件
- 发下载链接给用户

### 每卷完成
- 使用 lightclaw 上传整卷打包
- 发下载链接给用户

### 全书完成
- 合并为单文件（.md）
- GitHub 仓库备份
- lightclaw 上传完整包

## 导出命令

### 合并单文件
```bash

    /workspace/novel-factory/星际机甲崛起/创作/02-定稿/第1卷/*.md \
    ... \
    > /workspace/novel-factory/星际机甲崛起/星际机甲崛起-全本.md
```
